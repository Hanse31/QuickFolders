Description:   Opened: 2012-08-30 11:19 
Using SmartTemplate4 0.9.1 with Thunderbird 14 and 15, my template is always
inserted below the quoted message when replying. This didn't happen with
previous versions of SmartTemplate4.
My Thunderbird is set to compose messages in plain-text only and I also
disabled HTML tags in SmartTemplate4.

Reply template:
   %from% wrote:

Use HTML tags: unchecked
Disable default Thunderbird mail header: checked
------- Comment #1 From Axel Grude 2012-08-30 11:41:51 ------- 
Created an attachment (id=7379) [details]
Fixed version

Here is a version that behaves better. For the future, we have decided to split
up the quote part and the header part into 2 fields, which makes it easier to
place everything at the right position. The position of the reply can be set
from account settings > composition & addressing

select
[x] automatically quote...
    start my reply above the quote
    place my signature below the quote

smartTemplate will then put:

1. Template
2. Quote Header
3. Original Email
4. Signature

you can also choose
    place my signature above the quote

smartTemplate will then put:

1. Template
2. Signature
3. Quote Header
4. Original Email

Note that quote header is designed to be _always_ in front of the Original
Email - this is where you put "%to% wrote:"
