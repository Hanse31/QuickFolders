Well you were on the right track in your case you would have chosen the first
item Inspect Chrome Document - Inbox. (Inbox would have been the title of your
mail 3-pane window at the time. Then use the inspect button (top left) to point
to the DOM element you are interested in - in your case probably the tool icon
or its container box.

On the right hand side (where is says Object - DOM Node) you can drop down to
CSS Rules (which shows all rules that apply and their origin) or to Computed
Style (which shows the "winning" rules which apply to the DOM element and are
actually in force. Remember you can have multiple conflicting rules via
multiple style sheets and parent inheritance, the one with the highest
specificity wins)

You could check if there are warnings or errors in the error console, but
please do not attach screenshots of it. Instead, install console2 (a better
version of it) and copy the stream of errors as text via right-click.

After installing console2 go to tools > error console. There, enable errors,
warnings and messages; in the second row enable chrome and css, DISABLE
CONTENT!!!! 

In QuickFolders > Options > Advanced enable [x]Debug Mode and then right-click
Debug and enable the following details:
extensions.quickfolders.debug.events
extensions.quickfolders.debug.interface
extensions.quickfolders.debug.listeners.tabmail

Don't forget to disable content in error console.

Delete all contents of the error console before starting your test and capture
the output (using right-click, select all, right-click, copy) you can then put
the output into a text file and attach to this bug.

Oh, did I mention you must disable Content logging?
