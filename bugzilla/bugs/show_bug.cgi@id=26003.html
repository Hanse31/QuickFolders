Description:   Opened: 2015-04-05 06:37 
OperaMail automatically filters mailing lists on delivery, each into its own
folder. I would love to see this feature added to QuickFilters. The folders
have names that are automatically generated with lengthy alphanumeric strings,
but those are easy enough to rename. Those strings are unique to the LIST, not
the sender, which is very helpful to me, for example, since I need many lists
from U.S. government agencies -- multiple lists per agency. 

Here is what I was able to find about how Opera automatically filters mailing
lists:
> Mailing lists you are subscribed to are automatically recognized by Opera Mail, as long as the subject header contains an one of the following: 
X-Mailing-List, List-ID, List-Post, or Mailing-list in combination with
Delivered-to. 
See IETF RFC 2919 for specifications.  http://www.ietf.org/rfc/rfc2919.txt

That link gives all the specs for how mailing lists "should" operate, and I
must tell you that the built-in tool in Opera was pretty amazing in picking up
my lists -- about 50 of them all total, and a bunch I had forgotten about from
various addresses. It saved a ton of time making my own filter rules, and made
it very easy to unsubscribe.

And here is another standard I found: https://www.ietf.org/rfc/rfc2369.txt
And there is also this, but this looks like it is more applicable to ISP
http://dmarc.org/

Many thanks!

I don't think I can do something like generating automatic filters but you will
be able to create new filter templates specific to mailing lists, using the
feature described at [Bug 25989] (Support Creation of Custom Templates).

With the custom Templates you can add placeholder variables for all header
fields (such as list-id). You only have to add them from the "Customize..."
dropdown in the left hand dropdown in filter rules. You can then add them using
the syntax %headerName%  (e.g. %list-id%)

Once you have created the custom template, it will be available when you
generate new filters and so you can quickly define filters based on mailing
list specific mail headers, fully customizable for all sorts of tasks. I know
this is still a far cry from a fully automated system but it is a lot closer to
what you want than having to do it completely by hand
