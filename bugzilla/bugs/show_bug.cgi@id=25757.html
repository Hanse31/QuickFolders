Description:   Opened: 2014-04-24 11:12 
Is it possible when you move mail to another folder that it will automatically
be marked as read? (I'm not using the automatically mark as read feature).
------- Comment #1 From Axel Grude 2014-04-24 11:40:40 ------- 
The question with this (if you find this sort of thing useful) - should it
generally set anything moved to a Tab or submenu as "read" or should it be
controlled by a modifier key?

The current modifiers are
CTRL = force copying dragged mail to folder
SHIFT = jump into folder after moving the mail

I don't think I have any other ones at the moment.

'M' key to mark it as unread when I want to.
------- Comment #3 From Tony Mechelynck 2014-05-12 12:31:21 ------- 
Mmm…

When I get new mail into the Inbox, sometimes I read it before moving,
sometimes not. After reading (with auto-mark-as-read), I may move it with no
further ado (if it requires no further action) or mark it as unread then move
it (if it requires some action which I'm not ready to do now). IOW, an unread
message means to me something which is "not yet done with". When I move a
message without reading, it normally stays unread, the idea being that I'll
handle it when I come to that folder.

My Inbox is only a "transitory way station" where nothing ought to stay very
long.
