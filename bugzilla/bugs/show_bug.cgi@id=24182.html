Description:   Opened: 2011-07-07 03:45 
I can see no folders in the toolbar.  The only thing that is there is
"QuickFolders: [cog icon for options dropdown] Drag folders here to bookmark. 
Click the  ... ", etc.

I am not sure when this problem started, but I know I had a working set up a
while back.  

I have disabled all other addons and I still have the problem. 

I have tried uninstalling quickfolders, then removing its folder:
~/.thunderbird/arabg0t3.default/extensions/quickfolders@curious.be/
then deleting everything in my configuration that relates to it:
~/.thunderbird/arabg0t3.default/prefs.js
then restarting Thunderbird and reinstalling.  No joy.

I am using Thunderbird 2.0.0.24 (20100228) and QuickFolders 2.7.
------- Comment #1 From Axel Grude 2011-07-07 04:02:05 ------- 
Hi there, 

first of all, if you have the mnehny extension installed, this might be the
culprit - uninstall it. Next make sure you do not have any folders that have "
(double quotes) in their names. Next you can try the "tidy up" command (click
the cog wheel, click "Find orphaned tabs") - this will remove any invalid tabs.

You can also try to install the latest preRelease (1.7.1pre7) from
http://downloads.mozdev.org/quickfolders/

If these steps do not fix the problem, please install console2, go to tools /
Error Console. Then click on the "All" button and disable "Content". Restart
Thunderbird. copy and paste the error log immediately after start up.

PS: you are aware that Thunderbird 5.0 has just been released? Tb2 is getting a
little old at this stage...

------- Comment #3 From Axel Grude 2011-07-07 08:25:02 ------- 
thanks for the error log, pretty sure this one is resolved in 2.7.1pre4 or
later

here is the patch that fixed it:
(quickfolders-interface.js, line 471 ff.)

   var toolbar2 = QuickFolders.Util.$('QuickFolders-CurrentFolderTools');
+  if (toolbar2)
      toolbar2.className = theme.cssToolbarClassName;
