------- Comment #8 From Axel Grude 2012-11-29 15:16:12 ------- 
Created an attachment (id=7442) [details]
0.9.3 prerelease2

The problem is probably caused by the type of signature file being
misinterpreted as "text" although it is html. This new version has some special
variations on %sig% which can force either way:

%sig% - tries to determine whether there are html tags in it "auto"
%sig(html)% - always interpret sig as "html". 
%sig(text)% - always interpret sig as "plain text"

what you want to try is the second option.
