Description:   Opened: 2011-07-19 23:28 
In Postbox 2.5.x, when both Lightning (v. 1.0b3) and QuickFolders (2.7.1) are
installed, there is a bug which prevents switching between Lightning Calendar
and the Accounts/Folders pane. Once the calendar is opened in a new Tab,
clicking on the Inbox tab does not return to the default view. Disabling the
QuickFolders add-on fixes the issue. 

Screenshots:

Postbox window with QuickFolders installed: http://cl.ly/8adB
Postbox window with Lightning Calendar open: http://cl.ly/8adZ
Postbox window showing the display bug (note which tab is selected):
http://cl.ly/8bGn

Thanks!

Fixed version 2.7.2 Prerelease 1

Here is a fixed version, which works with Postbox 2.5 (tested in Windows XP).
Note there is a new option to hide the QuickFolders toolbar when it is not
needed (e.g. in a calendar tab).
http://img88.imageshack.us/img88/1751/hidequickfolders.png

This was a hidden feature in 2.7.1 but it caused the bug you described in
Postbox.

I haven't tested all permutations of this with Postbox (all different tab types
that should hide QF), so I would be happy if I got feedback on this and whether
it is regarded as useful.

PS: If you are a Mac user, I highly recommend trying the new Apple Pills style.
Looks especially cool when you chose single colors for tabs!

hide toolbar option

This was requested by a user and added as hidden feature in 2.7.1

The idea is to hide quickfolders when it cannot be used for dragging (e.g. when
calendar or a html content tab is displayed) the pros are freed up screen space
but among the cons is an annoying "jumping" of the content pane and mail tab
bar, so I tend to not use it most of the time.

I hear you. According to the error messages it appears to be a bug in
lightning, are you sure you are using the correct version for PB 2.1 ?

Try to go to about:config (Tools / Options / Advanced / General / Config
Editor) and toggle extensions.logging.enabled to true and see whether you can
see more error messages.

Also, if you activate and then right-click the Debug Mode setting in
QuickFolders options, you should try to activate the settings
extensions.quickfolders.debug.mailTabs
extensions.quickfolders.debug.toolbarHiding
this (plus checking debug mode) will add some informational messages that might
give some more clues as to what might be happening.
