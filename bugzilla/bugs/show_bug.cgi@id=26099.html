Description:   Opened: 2015-10-24 23:54 
It would be very nice to integrate also server-based, remote filtering by means
of the standardized sieve scripting languange. It is largely supported by
current IMAP servers, like Dovecot and Cyrus. Plenty of resources can be found
on the web, to get started in German for instance [1] and [2]. Remote access
should be established by the sieve management protocol, which is alreay used in
an existing TB-extension [3]. Thus it would be fine to have also this option
within quickFilters, to allow for remote filtering and synchronized rulesets on
different TB-installations.

[1] http://www.linux-magazin.de/Ausgaben/2007/06/Ausgesiebt/%28offset%29/12
[2] https://de.wikipedia.org/wiki/Sieve
[3] http://sieve.mozdev.org/

Thx,
Andreas

------- Comment #4 From Andreas Kemper 2015-11-02 01:30:13 ------- 
Hi,

I just did some research to find an easy solution for access to an IMAP-server
with sieve mail filtering included. My suggestion is to get the SOGo appliance
from http://www.sogo.nu/downloads/zeg.html and run it locally in VirtualBox. It
contains a properly configured Dovecot IMAP-server with sieve and managesieve,
running on standard TCP port 4190
(http://wiki2.dovecot.org/Pigeonhole/ManageSieve/Configuration).

The only thing you manually need to change there is the IP address for
managesieve from localhost (127.0.0.1) to your individual VM-IP. To do so ssh
into the appliance with sogo / sogo, sudo to get root access, edit the
corresponding /etc/dovecot/dovecot.conf and finally restart dovecot with
"service dovecot restart".

Hope this helps. If there are addtional questions please let me know.

Cheers,
Andreas

------- Comment #6 From Andreas Kemper 2015-11-02 03:02:38 ------- 
Hi Axel,

no, it isn't a big effort. Initially I also searched for some public
sieve-enabled IMAP-Server and thought about providing access to my own
IMAP-server.

But running the SOGo appliance locally is really easy, since it is completely
preconfigured and can be used right from the start. The only thing that need to
be adapted as said is the IP address for the managesieve service. In contrast,
you get full access to the IMAP-server allowing you to easily check for
instance logfiles.

Just download and install https://www.virtualbox.org/, import the appliance and
run it. In total it uses approx. 4 GB of space on the harddisk. In addition you
only need to ensure that virtualization support (VT-x / VT-d) is enabled in
your BIOS, since it is a 64bit OS inside the VM.

Cheers,
Andreas
