Description:   Opened: 2014-01-06 19:59 
I'm using Thunderbird 17.0.
I'm seeing a problem that happens most of the time, but not all of the time.
When an individual email is open and displayed in its own Thunderbird tab - if
I click on a QuickFolder tab, the display will not shift to that folder, it
will stay on the open email message.  If I then click on the Inbox Thunderbird
tab, clicking on a QuickFolder tab will work again, most of the time.
Strangely, if I disable QuickFolders, the Quickfolder tabs are still displayed,
and now clicking on them will always work again.
I've attached a screenshot showing what my Thunderbird tabs and QuickFolder
tabs look like.
Thank you.

------- Comment #6 From Axel Grude 2014-01-10 03:41:30 ------- 
Created an attachment (id=7766) [details]
2.14 beta 1

Try the attached version. I have decided to also release this as a beta on the
AMO site because I think it is pretty close to final release.

Essentially from a tab handling perspective this should behave the same way as
the version over at [Bug 25608]

Expected behavior:

1. If (while in Single Message view) a QF Tab is clicked that points to a
folder that is already open on one of the other Thunderbird tabs, switch to
that THunderbird tab.

2. If (while in Single Message view) a QF Tab is clicked that isn't open, then
open a new Thunderbird tab with a 3pane view showing the contents of that
folder.

if you have problems with it, please install the console2 addon and switch it
to "All" errors, then disabled "Content" logging and have the error console
open while you try to switch to another QuickFolders tab. there should be
warnings or errors in the log (these can be copied as TEXT via the right-click
menu).
