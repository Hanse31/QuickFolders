Description:   Opened: 2013-11-09 12:45 
Very small issue:

-- In my Reply and Forward quote headers, I use the expression:

%tz_name(1)%

This returns the text:

Sat 09-Nov-2013 8:38AM AUS Eastern Standard Time

It should be AUS Eastern Daylight Time.

In Australia, we changed to Daylight Saving Time on 06-Oct-13.

So, officially, the time zone in Sydney is AEDT (Australian Eastern Daylight
Time.)

In Date and Time on my PC, my time zone is set for:

(UTC +10:00) Canberra, Melbourne Sydney

with the box CHECKED for "Automatically adjust clock for Daylight Saving Time".

In summer (which is now in the southern hemisphere), Sydney is actually UTC +11
hours.

If SmartTemplate is "getting" the time zone from the Date & Time setup in
Windows, then it is failing to "see" that "Canberra, Melbourne, Sydney" time
zone uses Daylight Saving.

Thanks Axel for your attention to this so far.

Best regards / David

------- Comment #2 From dwallis@gmail.com 2013-11-09 14:15:00 ------- 
Thanks again Axel for your prompt attention to this.

The problem is almost like -

-- SmartTemplate gets its timezone from the setting in Date and Time in Windows

-- But Windows uses a lookup table that says:

  -- if you set the time zone to "Canberra, Melbourne, Sydney"

     ... and, if you check the checkbox that says "Automatically adjust clock
for Daylight Saving Time"

         ... then Windows has a lookup table that tells it to adjust the clock
FORWARD by one hour at 2:00am 06-Oct-13.  (That's for this year - it would also
include the DST dates forward and backward for the next 10 years - they are
gazetted that far in advance.)

So it appears that Thunderbird doesn't have a lookup table for DST on and off
for Canberra, Melbourne, Sydney.

Hope this helps.

Best regards / David.
