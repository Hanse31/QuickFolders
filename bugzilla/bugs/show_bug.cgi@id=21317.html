Description:   Opened: 2009-07-29 07:21 
In a certain installation with TB 2.0.0.22 and previous 2.0.0.x, the subfolders
displayed in the context menu of a folder tab appear in a "random" (but
constant) order instead of appearing alphabetically sorted. Happens with
QuickFolders 1.0.3 (and previous) as well as 1.2. It is not yet known what is
special in this installation. 

(Issue was communicated in more detail between user and Axel Grude.)

I have done some deep deep deep digging into the Mozilla source code and I am
coming to the conclusion that the GetSubFolders( ) function,  [which is
replaced by subFolders in TB version 3 and SeaMonkey 2], that returns a
nsISimpleEnumerator which iterates the (sub)folders - seems to be either buggy
or there is some hidden configuration somewhere that allows to change the
default sorting. Also have investigated aobut:config to no avail. There is a
small number of users affected, I can not reproduce the problem; when a new
folder is added in a session, it is appended to the bottom of the menu, but
after TB restarts, all folder menus are sorted alphabetically.

GetSubFolders calls a "dir" function further down in the hierarchy, but I did
not find any documentation about what methods can be used to assure it returns
folders in the desired (alphabetical) order. 

I have posted a query to the mozilla.dev.extensions group how to work around
the issue...

------- Comment #15 From Axel Grude 2010-03-22 03:48:09 ------- 
Created an attachment (id=6376) [details]
 QuickFolders 1.9 Prerelease 3

Added regexp replacements of some of the diacritics / umlauts and also
underscore (replacing it with /). This list is not complete but should cover
most bases especially for hispanic / french / scandinavian languages. Sorts:
any diacritical variation of a, e, i, o, u, such as { á,à,ä,ã,â,æ,
è,é,ê,ë, ì,í,î,ï, ò,ó,ô,õ,ö, ù,ú,û,ü }
ä and æ are treated as ae
ö is treated as oe
ü is treated as ue
Uppercase diacritics should be converted correctly (but no guarantee)

here is the replacement function
tr = {"\xE0":"a", "\xE1":"a", "\xE2":"a", "\xE3":"a", "\xE4":"ae", "\xE5":"ae",
"\xE6":"a",
"\xE8":"e", "\xE9":"e", "\xEA":"e", "\xEB":"e",
"\xF2":"o", "\xF3":"o", "\xF4":"o", "\xF5":"o", "\xF6":"oe",
"\xEC":"i", "\xED":"i", "\xEE":"i", "\xEF":"i",
"\xF9":"u", "\xFA":"u", "\xFB":"u", "\xFC":"ue", "\xFF":"y",
"\xDF":"ss", "_":"/"};
killDiacritics = function(s) {
  return
s.toLowerCase().replace(/[_\xE0\xE1\xE2\xE3\xE4\xF6\xF2\xF3\xF4\xF5\xF6\xEC\xED\xEE\xEF\xF9\xFA\xFB\xFC\xFF\xDF\xE8\xE9\xEA\xEB]/gi,
function($0) { return tr[$0] })
}

a slight performance penalty is expected, but I don't know how critical this
will be for people with lesser PCs (AMD X64 6000 here).
