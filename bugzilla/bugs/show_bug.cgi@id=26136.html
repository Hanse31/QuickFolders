Description:   Opened: 2016-01-13 10:17 
Two ways to access this functionality: 1. Double click a message in the list
pane, and 2. Right click the message in the list pane and select "Open Message
in New Window"

The first time after Postbox starts, this functionality works and the message
will be opened in a new window. All subsequent attempts to open a message in a
new window fail silently - with no indication to the user that the command was
received. After restarting Postbox, the "Open Message in New Window" command
will again work one time.

When the QuickFolders extension is disabled, this problem does not occur - in
other words with QuickFolders disabled, the "Open Message in New Window"
feature always works. 

All other commands/features in Postbox appear to be working normally. In
particular, the "Open Message in New Tab" capability works, and replying to a
message, which opens a new window to type the reply, continues to work fine.

This conflict in has been seen in Postbox 4.0.8 on Windows 7 Professional 64,
and in Postbox 4.0.8 on Apple OS X. The issue does *not* occur in Thunderbird
38.5.1 on Windows 7 Professional 64.

then turn off CSS logging. it is not important in this case. You see a large
amount of them in Postbox as Postbox is not yet fully CSS3 compliant;
apparently it gets some of the Thunderbird rules (which are written in CSS3).
The Graphics engine of Postbox  is a little outdated, but I am hoping that they
will eventually update to Gecko 14 or later (The current version is 44; postbox
uses version 9; that's why it still needs vendor-prefixes for many css rules)

> earlier today (before the colsole2 addon was installed), the console was also
> displaying Errors, Warnings, and Messages when "open in new window" was

Make sure you have JS and Chrome activated and Content disabled.

Chrome is basically the privileged layer where all the Javascript code of
Postbox and all Addons runs. Content is unimportant for this, it only helps to
track down html / layout errors in Emails.

You may have seen content errors. Or the CSS warnings flooded out the chrome
errors. So you might see different behavior once you disable CSS logging.

Also, you may try enabling the following options from the options dropdown
(right hand side)
[x] Report String Warnings
[x] Report all JS exceptions
