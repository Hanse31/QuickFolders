------- Comment #9 From Axel Grude 2013-09-29 04:35:51 ------- 
Looks like the bug in the log at least is caused by code from B2G (Boot 2
Gecko) which is used on Firefox OS devices:

https://bugzilla.mozilla.org/show_bug.cgi?id=895964

this is code that is likely (hopefully) going to be removed from Thunderbird in
the next release (24.0.1) 5 weeks from now.

Other than that I cannot really comment, as QuickFolders does not interfere
with the accounts or the way emails load - there is simply no code in QF that
does this. There are additional commands on the QF tabs (get mail) when they
are pointing to an inbox but these are additional and only get called when you
use the command from the context menu. The only thing I can advise at this
stage is to go to the Advanced settings and set a higher delay time (let's
5000) in the text box at the bottom right "[ms] init delay".

Also some POP3 providers might have a minimum time between logins so if you
restart Thunderbird only a few minutes after you started it, the get mail
command might get ignored by the mail server. (I have experienced this often on
my hotmail account). You might want to try and wait a few minutes before you
start up Thunderbird to avoid this being the issue.

When I restart Rb and look at my Activity Manager I see "2 messages downloaded"
 "No messages to download" etc. from the mail accounts that I have set to
automatically download mail at start up. If you want to help tracking this down
please install console2 and go to options > switch on strict warnings. Also
enable all messages except for Content.
