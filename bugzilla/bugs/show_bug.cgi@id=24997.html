Description:   Opened: 2012-07-06 06:05 
SmartTemplate4 0.8.5.6 on Thunderbird 13.0.1:

When creating a copy of a previously sent message using Thunderbird's "Edit as
New" function, the message header is copied correctly, but the message body is
sometimes missing; only the signature appears in the new message.

This problem never occurs if SmartTemplate4 is disabled.

It's not obvious why "Edit as New" works properly for some messages and not
others.  Messages quoting other messages seem to work.  Messages with
attachments seem to work.  Short messages with no special content (e.g.
hyperlinks) often do not work.  HTML versus plain text does not seem to be a
factor.  I haven't found a definite set of conditions that cause failure.

SmartTemplate4 settings:
   - Account: Common
   - Write: "Apply the following template..." NOT selected.
   - Reply: "Apply the following template..." selected.
            "Use HTML" NOT selected.
            "Use instead of default quote header" selected.
   - Forward: "Apply the following template..." selected.
              "Use HTML" NOT selected.
              "Disable default Thunderbird quote header" selected.
------- Comment #1 From Axel Grude 2012-07-06 07:21:00 ------- 
The question is whether SmartTempalte4 should make any changes whatsoever as
"Edit as New.." is really designed to edit a message as it was, without
changes. This is all the more important as I can use the command multiple time
on a message that is saved in drafts; it wouldn't be productive to run the
modification code again and again.

So I propose that "Edit as New" should do the same as opening an Email from the
drafts folder: open the Email unmodified so that it can be edited as it was
originally.
------- Comment #2 From Scott Inrig 2012-07-06 07:32:14 -------
