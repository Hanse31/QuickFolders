------- Comment #3 From Axel Grude 2019-11-20 05:12:20 ------- 
To inspect THunderbird open Tools > Developer Tools > Developer Toolbox (or
press CTRL+Shift+I)
Click the Inspector tab, and the "Picker" tool on the top left.
Click on the gray area, making sure it is highlighted with a red dashed line
around it. If you hit a child element and can't access the exact element, you
can use the breadcrumb tree at the bottom, e.g.


rows > row#CompatHeader_collapsed2LtoCcBccRow >
mail-multi-emailheaderfiel#CompactHeader... > hbox.headerBlueBox >
description.headerValue

you can click any of these items to highlight the appropriate element

On the right hand side you will see tabs for
- CSS Rules (most likely containing the reason for behavior either in the
element or a parent or a sibling element. Finding the correct one is often
difficult. It sometimes helps testing by adding new rules into the element{ ...
} are on top, such as min-height / max-height etc.)
- Layout  (shows box model = dimensions and margin / padding)
- Computed Styles ... shows (mostly) the rules that are actually applied and
maybe rules that are overriden by others.

Note, you may have to enable "Show Browser Styles" in global settings (three
dots top right) in order to see all style rules that Thunderbird applies.

For me the most important information is
- which element is the gray area
- what kind of box model valus has it got (height / width / paddings / margin)
- What's its parent? And what's the parent of the parent.
- does it have a sibling before it or is it the first Child?
- what's the next sibling (if any)

You can't "save" positions, they are determined by CSS rules. We need to find
out what rules cause this to be broken on your system.
the #messagepaneboxwrapper element should go from the top of the header to the
bottom of the preview box. In your case it seems to be "shifted up".

It contains a child element #messagepanebox which again should cover the same
area. 

Within that there are a few children, normally you should see only the
#singlemessage (unless you have multiple emails highlighted)

#singlemessage contains the #msgHeaderView (header area) and the
#messagepanewrapper which is the mail preview area. underneath the preview area
are a few elements that are normally hidden, such as #attachmentView,
#attachment-splitter, #FindToolbar

do any of the three above (messagepanebox,singlemessage, messagepanewrapper)
extend to the bottom of the main window? Or are they all shifted "up"?
