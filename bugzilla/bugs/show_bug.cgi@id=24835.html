Description:   Opened: 2012-04-06 11:15 
On TB 3.1.9pre running on Linux, tab appearance customization in QuickFolders
3.0.2 doesn't work. Any settings on the "Bling My Tabs!" options tab has no
effect.

The tab bar always appears as a continuous, flat, gray bar with the text of the
folders appearing with no visual separation between the folders. The number of
unread messages appears after the folder name, but there is no visual
indication of unread mail.

The Tab Color context menu shows "No Color", and then "Color 1" through "Color
20", but they all appear in the same system theme gray. Selecting a color has
no effect.

If I go into options and select a different tab appearance style, then hit the
shortcut to rebuild the tabs (not sure if that's necessary), noting appears in
the error console. (There is a warning upon entering the settings dialog:
"Warning: Expected number or percentage in rgb() but found 'NaN'.  Error in
parsing value for 'background-color'.  Declaration dropped.  Source File:
chrome://quickfolders/content/options.xul  Line: 0")

On Linux, you should run thunderbird (with no extension) which is a script
which invokes run-mozilla.sh, a further script which finally invokes
thunderbird-bin (possibly after setting some environment variables etc.).

On Mac, you should drag thunderbird.app out of the .dmg onto your, er, desktop
or whatever it is called on the Mac, and run it from there.

On Windows, running thunderbird.exe directly should work, but IIUC there is
also a thunderbird.bat

Yeah, I overlooked the call to run-mozilla.sh. run-mozilla.sh is in fact
necessary to set up the library paths. Running without it results in:

/usr/lib/thunderbird-3.1.9pre/thunderbird-bin: error while loading shared
libraries: libldap60.so: cannot open shared object file: No such file or
directory

So if you are running thunderbird-bin directly, this is probably why it is
failing.
