works fine until dragging an email to a Quickfolder. Dragging an email to a

dragging to QF again.

This has been a consistent problem for us on all 4 computers for at least a
couple of years.

LOVE QF otherwise!

First of all, try it without QuickText pro to see if that makes a difference.
Then I would suggest to use console2 to generate an error log to see what
happens 

1. at the end of the drag & drop to a QF tab
2. when you right-click on the QF tab after the drag operation

# In QuickFolders options, enable Advanced / Debug Mode
# Right-click Debug mode and switch on
  @ extensions.quickfolders.debug.dnd
  @ extensions.quickfolders.debug.folders.select
  @ extensions.quickfolders.debug.mouseclicks
  @ extensions.quickfolders.debug.popupmenus




Install 
https://addons.mozilla.org/en-US/thunderbird/addon/console%C2%B2/

Open Tools > Error Console
Activate [all]
Deactivate [content]  - very important to avoid spamming the log with
unimportant stuff like errors badly formatted emails

Hit [Clear] and do the first test. You can use the Error Console right-click
menu for Select All / Copy to copy the log to this bug.

Hit [Clear] and do the second test. Copy log as above.
