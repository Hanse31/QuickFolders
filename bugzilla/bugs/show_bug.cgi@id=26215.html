Description:   Opened: 2016-04-28 14:59 
In Thunderbird v45.0 I turned on in Thunderbird preferences the "When using
paragraph format, the enter key creates a new paragraph" option. I use
SmartTemplate4 for REPLY, with nothing typed into the "Template" section; I
only use the "Quote Header" section. If I do a Reply, the email starts
correctly in Paragraph mode. But when I type the first letter of the email, the
email a) changes from Paragraph mode to "Body Text" mode and a blank line is
inserted between the first line and my signature.

Axel Grude suggested a workaround of adding <p>%cursor%</p> in the "Template"
section of my reply. This solves my problem, but he suggested I add a bug
report so this solution may be automated.

SmartTemplate 1.3 prerelease 47

Added some more improvements when creating the <p> flag around the cursor
position. If the cursor position is already in a paragraph no special action is
taken. Otherwise a new <p> tag is started by backtracking to the last block
closing tag (</div> or </p>) or <br> and start the new <p> from there. 

The new <p> tag is then closed when encountering the next opening block element
(<p> or <div> again).
