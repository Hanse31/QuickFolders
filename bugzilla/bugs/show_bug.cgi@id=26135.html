Description:   Opened: 2016-01-11 17:36 
Suggest providing a "Move To" button in the main toolbar that allows the user
to move email to a folder quickly and easily by selecting the destination
folder from a short, inline dropdown list.

This enhancement is essentially to implement Gmail's well designed "Move To"
button, with the addition of integration and user configurability suitable to
enhance QuickFolders. (The Gmail "Move To" button appears at the top of the
screen, above the email, when you're reading an email in Gmail).

Both Thunderbird and Postbox already contain a "File" button, which provides
the ability to move files to a folder. However the interface for it is not
streamlined like the "Move To" button. As it is currently implemented, the
"File" button is too cumbersome to use like a "Move To" button on a regular
basis. The default installations of the email clients don't even place the
"File" button on the toolbar. However, with a little tweaking and user
configuration it could become a superstar of user convenience for filing
emails, and provide an additional convenient workflow for users to manage their
emails using QuickFolders. 

The enhancement doesn't necessarily need to be implemented by modifying the
File button, though building on the existing functionality of the File button
may (or may not) be the better approach.

Some suggested upgrades, when viewed as an enhancement to the existing "File"
button in the main toolbar:

1. Allow the user to configure the File button to specify:
a. What folders it will display.
b. The order that the folders are listed.

2. Allow the user to modify the File button so that the list of displayed
folders are displayed inline - in a single column, rather than (the current)
nested hierarchy that requires the user/mouse to move to the right, search
through sublists, and navigate the entire folder structure in order to find the
one they're interested in. That is often not necessary, and makes the File
button too cumbersome to use for typical users who file their emails to a small
set of folders.

3. When the file button is clicked, in the drop down list that displays the
file names, display each file name highlighted by the color of the
corresponding folder (Applicable whenever the user has assigned a color to the
folder on the QuickFolders bar).

4. For users with multiple accounts or for other use cases that might benefit,
allow the user to instantiate more than one "File"/"Move To" button on the main
toolbar (with unique names, icons, and/or colors), in order to provide quick
access to the folders they use most.

so well, then this enhancement is clearly beyond its scope. If the purpose of
QuickFolders comes to include providing quick folder access in the separately
opened email windows and/or providing quick folder access in the main window
more generally, then this would be a good enhancement to consider. It is a well
designed interface with legions of Gmail users who would be happy to have it
for quick folder access.
