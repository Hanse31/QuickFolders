Hi! - QF 2.9.3 on a PC running Kubuntu 11.10; with recent TB 10.0.2 update,
default Theme, all other extensions disabled.

[attached image is two screenshots in one image; sorry for that, but it only
allows me to attach one file]

When I go to a folder, only the first four messages are shown, even though I
have many messages in the folder (I can scroll to see them, but the window size
is limited to just four). [see top half of attached image]

If I pull the divider for the message display pane all the way to the bottom
(shrinking the message-viewing pane to nothing), the message list window grows
to show a few more messages, but not nearly as much as it should... something
in the algorithm is obviously wrong. [see bottom half of attached image]

Similarly, the left-side pane showing the various accounts and their folders
isn't sized correctly: it seems to have the opposite problem: too many are
shown for the actual window size, and the window is cut off by the pane [top
half of attachment]. If I shrink the message viewing pane to nothing, the
account viewing pane seems normal [bottom half of attachment].

The accounts panel works OK when it has full run of the window, and when the
thunderbird window is maximized, but is otherwise wonky, as described.

I am using View -> Layout -> Wide View, but similar issues exist in all three
views. Let me know if you want pictures of the other view types (same thing
with slight variations: message list too short, account list too big).

I have no user chrome in effect.
