Description:   Opened: 2017-10-13 01:45 
How to reproduce (windows 10 and Thunderbird 52)
1. Use an IMAP account
2. In account setting, Cpies & Folders, set "Place a copy in "Sent" Folder on:
xxx
3. In account setting, Synchronization & Storage, set "Keep messages for this
account on this computer".
4. Take an e.mail from your sent folder and select Reply
5. You will notice that the message header is the standard Thunderbird header
and not the SmartTemplate4 one
6. From that moment on, Thunderbird will not be able to access your sent folder
anymore. E.g. if you send an e.mail , it will not be able to copy it in the
sent folder.
7. If you close Thunderbird, you will see the crash reporter window of
Thunderbird, which let me think Thunderbird is crashed in quitting down.

Similar issue happens with forward of a sent message
