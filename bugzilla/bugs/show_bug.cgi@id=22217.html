Sorry, but that is just not (easily) possible (don't even know if it is

thrust of development for the next releases will be to focus on better support
for the smart / search folders. Once this is done, integrated Icon support
might be a good area.

That's cool to hear! Max width would of course have been possible as well (but
the strings would have been truncated, not nicely abbreviated like in the
folder pane) - all you need is a little rule in userChrome.css:


#QuickFolders-Toolbar.toolbar-flat toolbarbutton {
  max-width: 50px !important;
}

you could also have icons for each individual tab, based on the name (16*16 pix
works best):

#QuickFolders-Toolbar.toolbar-flat toolbarbutton[label^="Paypal"] {
    list-style-image: url("QF-Icons/paypal.gif");
}

(you would have to create a folder QF-Icons in your chrome folder, and have the
icons in that. the "label^=" rule matches on the start of the tab name, in this
example, Paypal); in combination with the max-width rule you could even omit
the text altogether.

I will have to add a "QuickFolders tweaks" page to the Mozdev pages soon!

Sure, this is possible:

#QuickFolders-Category-Box {
    margin: 0;
    padding: 0;
    vertical-align: center !important;
    overflow: visible;
}

    max-height: none !important;
    margin: 0 !important;
    padding: 0 !important;
}

As you see, with CSS just about anything is possible.

> Oh. I can change the color of the black line by changing default tab colors :-)

Yes, its not all that obvious, but by default QF uses the highlighting colors
of your OS theme (usually white on blue) which corresponds to the appearance of
your Active Menu Items. The colored line is supposed to connect with the active
tab to add a more "tab-like" feel.

I guess to make it less intrusive you could try something like

#QuickFolders-Toolbar.toolbar-flat {
    border-bottom-width: 1px !important;
}

in your userChrome.css. Or use 0px to make it invisible.

You can change about any visual aspect of Thunderbird by editing that file. It
takes a little bit of knowledge but you can find out what the screen elements
are named by using the DOM inspector to make up more rules.

e.g. transparent background (to use the default for toolbars):
#QuickFolders-Toolbar.toolbar-flat {
    background-color: transparent !important;
}

without getting in too much detail here the # is for element's Id, the . is for
classes.
