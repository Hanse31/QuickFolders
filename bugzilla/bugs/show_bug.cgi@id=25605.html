Description:   Opened: 2013-10-09 09:58 
QuickFolders: 3.12.1

I have two folders that I created in Thunderbird called 'ESales' and 'Archive'.

'Archive' folder had two sub-folders within it. I accidentally pulled one of
the sub-folders out of 'Archive' into the QuickFolders Toolbar.

I wanted to remove newly created QuickFolder tab. However, when I right-clicked
the context menu of the new QuickFolder tab, the only option was: 'Open in New
Tab'. The other context menu options were missing. 

Since the other context menu options were missing, I could not use the 'Remove
Tab' option that would have removed the QuickFolder tab from my QuickFolders
Toolbar.

Axel helped trouble shoot this issue with me from October 7-8th. The Error
Console in Thunderbird was not reporting any issues with the add-on. The 'Find
Orphaned Tabs' option in QuickFolders did not fix the issue. There was no
chrome.CSS file or custom CSS styles linked to this Thunderbird account's
add-on.

Axel's solution was to reset the 'QuickFolders.folders' string, which was done
via the Config Editor in Thunderbird. This fixed the issue by resetting my
QuickFolder tabs so I could start from scratch again.

Axel suggested I created a bug report here in Bugzilla on October 9th.
------- Comment #1 From Axel Grude 2013-11-18 04:54:38 ------- 
I cannot reproduce this behavior. Any folder that I have dragged to create a
new QuickFolders tab, has a context menu with an "QuickFolders Commands" sub
menu; are you missing the QuickFolders Commands sub menu itself?

I also tried creating a folder that is in a special "Archives" folder. Maybe
you shouldn't create a Folder called "Archives", or make sure that your account
settings point to it:

right-click your mail account in folder pane, select properties
click on Copies & Folders
Enable [x] Keep message archives in: ...
Click on Message Archives [Archive Options...]
Select an option and click Ok. In my case "Yearly archived folders" I just see
a folder for each year that I have archived. It is possible that other folders
in the "Archives" folder are somehow not visible to Thunderbird (?) If you want
to have your own folder structure in there you could try renaming the
"Archives" folder to something else.

You can also repair the folder string via the about:config window, but it is a
little complicated as it is one long JSON string (without any line breaks). You
would have to open tools > optiopns > advanced > general and click on [Config
Editor...] and then enter 
QuickFolders.folders
in the Search box.

A folder entry looks like this:
{"uri":"URL","name":"NAME","category":"CATEGORY","tabColor":"NUMBER"}
and are concatenated and delimited with Commas

in your case your would look for something that looks like
{"uri":"xxxxArchivesxxxx" ... },

you would have to delete this whole string including curly braces and trailing
comma and then save the changed string. Then immediately restart Thunderbird.
