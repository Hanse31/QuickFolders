Description:   Opened: 2013-11-28 10:37 
Thunderbird's new built-in font size shortcuts CTRL+< and CTRL+> are triggered
by certain ZK combinations. Make it possible to disable these.
------- Comment #1 From Axel Grude 2013-11-30 15:08:06 ------- 
Created an attachment (id=7740) [details]
2.16 prerelease 22

This took a fair few hours as I noticed that this again locale dependant (so it
depends on your keyboard language on whether this has an impact). I made a
generic function that matches the keyboard accelerators of the editor against
that active in the current zombie keys locale; this also made it necessary to
add "alternative" keys to all mappings of all locales, but the good thing is it
should be future-proof even if Thunderbird decide to add more key shortcuts in
the future: these will all be detected (if my mappings for the locale are
correct) and removed. I tested it on the IRL keyboard (so should work also on
UK locale). I mapped all other locales based on the key layouts on

http://zombiekeys.mozdev.org/layouts.html

so it _should_ work in other keyboard layouts as well - provided the correct
layout is selected in Zombie Keys > configuration. Please test in your own
locale.

For full debugging, or in case of problems, I recommend activating debug and
the setting
extensions.zombiekeys.debug.accelerators = true
extensions.zombiekeys.debug.defaults = false
(to configure this, right-click the debug checkbox in Zombie Keys Options
window) - this will give details on all attempts to remove accelerators during
opening of the Editor window. (In Tools > Error Console)
