Description:   Opened: 2013-10-11 04:48 
When dragging the current email(s) by using the little envelope icon on the
left hand side of the current folder bar (above message preview), the drag
operation never completes - the drag hand stops existing once I leave the
button area.
------- Comment #1 From Axel Grude 2013-10-11 10:04:04 ------- 
Created an attachment (id=7705) [details]
3.12.2 prerelease 17

Fixes dragging to tabs and folder tree from the envelope. You can also drag
emails to the desktop (windows explorer) - at the moment it only works when you
drag one email, I still need to find out what is wrong with that, as it is
possible to drag multiple mails from the messages list. This was broken by a
fix to [Bug 812923] Thunderbird.

I raised Thunderbird's minVer to version 17.0 for this because it uses the
Set() class.


Also, it is still broken in SeaMonkey.
