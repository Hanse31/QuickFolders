Description:   Opened: 2009-09-19 14:50 
I have not quantified this, but I'm fairly sure that TBird respone to message
moves from one folder to another is lagging greatly when QuickFolders is
enabled. 

Disabling QuickTabs in Tools|Add ons seems to eliminate the lag (after program
restart, of course). 

I have not clocked the difference, as it seems to vary quite a bit. Perhaps
over a minute in some cases.

This occurs both for message moves effected by drag and drop from the message
list pane to a folder in the side pane, keyboard Alt+M, M, etc, and drag from
message pane to QuickTabs folder tab. 

Machine here is WinXP srv3 running on Intel Celeron at standard clocking on a
(yes, elderly) Jetway P4M266-8235 board with 1024 MByte PC2100 Corsair DDR RAM
(2x512 MByte). The boot drive is an  250 GByte Seagate PATA running at 7500rpm.
Second internal drive, same specs. 

Figured I'd best report it!

------- Comment #3 From Axel Grude 2009-09-20 01:45:22 ------- 
The bug is caused by multiple messages being sent to the folder listener when
multiple messages are moved - the update of the UI can take quite some time so
there is scope for also reducing the overhead. I was aware of this problem and
had added a timer as fix for the planned release of 1.3, so that the update of
the number of total items / unread items is executed asynchronously - this will
both remove the lag and free up performance. 

I also do try to avoid the UI Update if it is not necessary (e.g. total unread
folders is not displayed).

By design, this will introduce a lag of the numbers update on the QF folders
(and submenus) but I think this is minor compared to the performance boost for
Thunderbird.
