SeaMonkey/2.2a1pre

Reproducible: Didn't try

Steps to reproduce:
1. Install QF 2.5 after not having used any recent version of QF

Actual results:
At first startup after install of QF 2.5, a dialog pops up, asking for an
"application", with Firefox as default and one other choice ("Browse..." or
equivalent, I'm quoting from memory).

Expected results:
On SeaMonkey, the URL for an HTML page should be directed to the SeaMonkey
browser even if it is not the system's default browser: this is what
Sm-MailNews does when clicking a link, and it is the expected "Suite" behaviour
(inherited from the Mozilla Suite I suppose, and perhaps even from Netscape
4.x).

Additional info:
If I browse to a softlink (in my case, /usr/local/bin/seamonkey ->
../seamonkey/seamonkey), the dialog pops up again. After selecting
/usr/local/seamonkey/seamonkey the HTML page appears in the SeaMonkey browser.
