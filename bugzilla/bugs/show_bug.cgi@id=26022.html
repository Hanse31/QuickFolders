Description:   Opened: 2015-05-14 19:03 
Creating filters by dragging to folder and saving does not persist between
Thunderbird restarts. I save the filter and it appears in the list of saved
filter rules.  I exit Thunderbird and then restart it and go back to my filter
rules and they are gone.  Can you please advise what could be causing this. 
Are there limitations on the number of filter rules in a list that can be
saved? quickFilter version 2.8.  Windows 7, Thunderbird 31.6.0

> Creating filters by dragging to folder and saving does not persist between

> filter rules.  I exit Thunderbird and then restart it and go back to my filter

Some invalid rule, maybe? Check your msgFilterRules.dat after you have created
the filter and closed the filter list. Apparently Thunderbird saves the file at
that stage.

> Are there limitations on the number of filter rules in a list that can be
> saved? quickFilter version 2.8.  Windows 7, Thunderbird 31.6.0
> 

I don't know, as I do not store the filter rules, this is Thunderbird code. If
you check your msgFilterRules.dat you will probably find the rule and you can
attach it to the bug. A filter rule starts with the "name" entry and looks like
this:

name="Addons - Menu On Top"
enabled="yes"
type="17"
action="Move to folder"
actionValue="mailbox://xaxax@axaxax.ie/Development/Mozilla/Addons/Menu%20On%20Top"
condition="AND (subject,contains,Menu On Top) OR
(from,contains,xyz_xxx@mac.com)"

------- Comment #3 From Axel Grude 2015-05-15 11:03:40 ------- 
(In reply to comment #2)

> > the filter and closed the filter list. Apparently Thunderbird saves the file at

> I tried it with all my addon's disabled and was able to save and persist a

If the filter is still there when you open up the filter list after closing it
then it isn't a quickFilters issue. qF generates the filter in memory
(programmatically) and then lets the user edit it and lets the filter rules
dialog (Thunderbird) save it in the list.

(In reply to comment #2)
> After I save and exit the filter list I check the msgFilterRules.dat and the
> rule is not in the file.  In fact something perplexing is that there seems to
> be only 4 rules in the file when I should have a lot more -- at least 20-30.
> 

Could be the "wrong one". There is one for each account, including the ones in
ImapMail. locations can be deceptive as the email might have been (as is often
the case)  moved across to a different account (by your filters_). When in
doubt look at the recipient address and find the appropriate mailbox. Most
likely it is _not_ the file in "Local Folders".
------- Comment #4 From Lui 2015-05-16 16:44:25 ------- 
I deleted the msgFilterRules.dat file from the folder for the Account that I
was having issues for and re-saved all my filters -- select all --right click
quickFiler create.  This added the rules back and persisted between TB
restarts.  Messages are being filtered properly -- issue resolved.  I guess
invalid rules in the msgFilterRules.dat file cause TB to not save the any new
filters.  Thx for your  help.
