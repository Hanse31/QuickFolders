On the desktop side I am running Ubuntu 12.10 (Linux version 3.5.0-21-generic
(buildd@roseapple) (gcc version 4.7.2 (Ubuntu/Linaro 4.7.2-2ubuntu1) )

to the local address with each account.

When I create the filter manually it works fine.

Any other information that is needed I will be happy to help out.

------- Comment #4 From Axel Grude 2013-04-15 01:30:52 ------- 
(In reply to comment #3)
> I have the same problem. Works perfectly for filters to folders within the same
> IMAP account, but any filters to move an email from the Inbox one account to a
> folder in another, throws this error.
> 

Do you also get 

No targetFolder.msgDatabase!

this is the core of the problem. The first thing I would think is that GLODA is
not enabled for that account.

Account > Settings > Synchronization & Storage
[x] Keep messages for this account on this computer 
[Advanced...]
select all folders you want to filter for offline use

remember that the filter runs on your local machine, not on the IMAP server.

As a workaround, you can also try to right-click the message _after_ you have
moved it and select 
QuickFilters: create Filter from Message...

this will try to determine the original inbox it has been moved from after the
fact; it might be easier for Thunderbird to come up with the info on the target
folder once the message has successfully registered in the global database
(GLODA).
