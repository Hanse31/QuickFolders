------- Comment #13 From Axel Grude 2011-07-24 11:57:32 ------- 
Created an attachment (id=6996) [details]
This avoids the call when not necessary

Although this is a Thunderbird bug, I can avoid the call to
GetFirstSelectedMsgFolder by checking the existence of gFolderTreeView
beforehand, so I was able to avoid this problem (*). To really get this fixed
raising a bug in Thunderbird itself might be the correct cause of action. Stack
Trace reports usually are more about the platform (Gecko / NMozilla / FIrefox /
Thunderbird / Postbox) itself, and errors in extensions typically will show up
in Tools / Javascript Error Console.

(*) Note: this fixes the problem exclusively in Thunderbird 3 and later. It
does not apply to postbox / SeaMonkey or TB 2.
