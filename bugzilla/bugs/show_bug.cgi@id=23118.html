------- Comment #4 From Axel Grude 2010-08-13 13:10:54 ------- 
What does the error console say? 


I have tried reproducing the issue with TB 2.0.0.24 on XP SP3, but no joy -
works fine. All tabs are visible. Are you using Folder Categories? If this is
the case try the categories "(Display all)" and "(Uncategorized)".

When adding an existing folder to the QF toolbar it should switch to its
category and activate the tab automatically. There were no major changes in the
way that this behaves...

Also, try to increase the setting [ms] init delay on the first option screen to
8000.

When using error console, you have to enable strict, warnings, errors, and
chrome, disable content to get the best result. For an easier way to see errors
try to install console2 - I have instructions and a link on the page:

https://quickfolders.org/faq_tips.html

look at the question "Something is not working, I think there is a bug, what do
I do?"

With console2 you can catch all the error output and copy it into an email. In
the meantime, I do another test on linux.

Axel
