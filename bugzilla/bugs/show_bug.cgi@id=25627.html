Description:   Opened: 2013-11-07 07:52 
Dear Axel,

as I wrote, here a request for the Gimmick "Group of People".

The Idea :
Select different eMails with different eMail-adresses, after clicking on your
quickfilter-assistant and then pull it onto the corresponding folder (like the
way it is in the currently addon).
The filter recognized now all adresses and create a corresponding filter, not
only for one emailadress, but for all marked emails (then maybe that works with
the selection criteria specified in the quick filter).

You answered me on Nov. 7th. and told me to make an request, so I do.

Andreas Griebl

Good idea.

At the moment quickFilters only looks at the first email when you drag multiple
ones. Of course it is a little tricky... there are a couple of things that need
to be added, like a "stop-list" for certain addresses (my own addresses) as I
wouldn't like any of my aliases trigger the filter if I dragged a conversation
to a folder that contained replies by myself. Also I presume it would only work
on "From" Addresses (not CC or To) like the "based on Sender" Template. I think
ideally this would be a new template (will attach a mockup screen shot for
discussion next)
------- Comment #2 From Axel Grude 2013-11-07 10:36:09 -------

UI mockup

This is what the interface of the new template could look like. It should
probably only be enabled if multiple emails are dragged, although there is no
harm allowing it with a single mail (edge case).

I suggest keeping this separate from the existing templates as it is purely
specialized in harvesting "From" addresses. Also it needs configuration to
avoid adding certain email addresses (which could be my own ones). It might be
best to have an additional "ignore list" that could be configurable via the
options dialog.
------- Comment #3 From Axel Grude 2013-11-11 08:08:47 ------- 
Created an attachment (id=7725) [details]
2.1 prerelease 2

A test version which has the template.

1. only English and German locale are added currently. if you have another
locale, this will not work for you (as nothing is translated and the UI will
throw an error)

2. The new template 'Group (senders of multiple mails)' can only be used when
dropping 2 or more emails.

3. Should work via context menu as well (not tested!) if emails are already in
the folder

4. I only looks at the "from" addresses (ignores all to, cc, bcc headers etc)

5. All (default) email addresses from the users own mail account are
automatically removed [as it wouldn't make sense to add them if you create more
than one filter this way]. This applies to all accounts (I believe even
inactive ones)

To test drag multiple mails into a folder during filter mode. 

For additional information, enable debug mode (quickFilters Preferences >
Options)
[x] Debug
right-click the debug mode checkbox and set 
extensions.quickfilters.debug.template.multifrom = true
Look at tools > Error Console
