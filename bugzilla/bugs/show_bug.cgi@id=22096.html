Hi Lars,

that works on my system, but there is a time delay on it (about 1 sec) - older
versions would do this straight away which could lead to enormous waiting times
(especially when moving multiple emails to another folder)

To see whether the timer works, you can activate the Debug checkbox in the QF
options and then go to the error console / javascript console. Delete all
listings and then toggle one or more emails to UNREAD in one of the folders
that s listed on a popup menu. You should get only "informational" messages,
one of them would be looking like this:

QuickFolders 
qfFolderListener: TotalUnreadMessages

this is effectively the event that Thunderbird sends if a folder's unread
messages count is modified. If you can not see that anywhere it might be a bug
of another plugin that hooks into the folder events. If you see any error
messages please send them (and the previous information messages) as an
attachment.

Also, can you see the number of unread emails after the folder name in the
submenu entry? IT works for me here (Windows XP).

thanks for your help
  Axel

I understand your dilemma :) the only advice I can give you at the moment is to
use the "move to the next/previous unread message" buttons, that will jump into
the sub-folder when it has mail. Alternatively you could make a stored search
folder that includes a recursive search into a QuickFolder.

Go to Edit /Find / Search Messages, select the top level folder then click
"Save as Search folder". In the "New Saved Search FOlder" window, select "Match
All Messages". Once the Search folder is created, pull it to the QuickFolders
bar. The only problem with this approach is that a Search FOlder does not
"contain" any Subfolders so there is no subfolder menu underneath. Also
threaded mail view does currently not work in a Search Folder.

But you could certainly use it to be alerted to and read new emails.
