Description:   Opened: 2009-10-23 11:47 
In certain Linux environments, such as Ubuntu Jounty when the users drag a
message into a quickfolder tab and then try to move it to a subfolder on the
subfolder popup menu, thunderbird just closes.

This bug happens during getAnyTransferData, its strongly suspected it might be
a bug in the Linux code base? I do not know whether there is a workaround that
avoids getAnyTransferData ...
------- Comment #1 From dGo 2009-11-06 06:13:57 -------

------- Comment #2 From Axel Grude 2009-11-08 15:08:22 ------- 
I have installed a test environment of Ubuntu 9.04 and spent some hours
debugging this problem. It seems Thunderbird under LInux does not like the call
that removes the popup menu (hidepopup) at the end of the drag operation.
Unfortunately I don't know how else to get rid of the popup after the drag. I
have built a slightly more stable version that omits this code (careful if you
drag into the menu and then back to the tabs you will still crash TB). At least
like this the feature of moving mail into subfolders is usable.
