Description:   Opened: 2014-07-21 10:48 
Right now it is not possible to use more than one template per account. It
would be great if you could choose which template to use when writing a reply,
regardless from which account.

For example: most of the times it is sufficient enough to use the standard
quoting style that has been used on the net for decades. But sometimes I need
to use the style that Outlook uses, because the recipients can't deal with any
thing else that looks different. 

So, a list of templates that you can pick one from when replying would really
be great. 

Also there should be one variable that cause the ">" to be deleted in the
reply.
------- Comment #1 From Axel Grude 2014-07-21 11:48:51 ------- 
> Right now it is not possible to use more than one template per account. It
would be great if you could choose which template to use when writing a reply,
regardless from which account.

At the moment this is not planned because it is very fundamental and st4
templates are tied to identities (*), but you can use the Stationery Addon to
do this: Install Stationery and activate the "Stationery Support" option. You
can then use html files and drop down the selection from the reply / forward /
compose buttons. You can use all smartTemplate4 variables inside them.

By the way you should still be able to write HTML templates with plain text,
just use <br><br> for line breaks.

> Also there should be one variable that cause the ">" to be deleted in the
reply.

Please do not make "combined feature requests" as these are hard to manage.
Please raise a separate bug for this.

(*) you can also create multiple identities within the same account to have
more than one smart template for the same account, but it is a little clunky as
you might have to change the identity from the composer window (by using the
dropdown on the from field)
