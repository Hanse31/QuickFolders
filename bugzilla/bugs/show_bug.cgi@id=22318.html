QuickFolders 1.8.5 Prerelease 8

This version tries to deal with a problem that might be especially affecting
users of the nl/NL locale. When adding a folder, a Message that is supposed to
warn about an already existing folder makes the program crash with the error
message:

Fout: Component returned failure code: 0x8000ffff (NS_ERROR_UNEXPECTED)
[nsIStringBundle.GetStringFromName]
Bron bestand: chrome://quickfolders/content/quickfolders-model.js
Regel: 20 

I have tried to work around this issue by catching the error and recompiling
the language file. If it still fails, an English Message is displayed instead.
More importantly, it will not stop the rest of the code which makes the folder
visible by selecting it and its Category.

------- Comment #8 From Axel Grude 2010-01-25 15:59:14 ------- 
Created an attachment (id=6243) [details]
QuickFolders 1.8.5 Prerelease 9

Found that the language files in the nl/NL locale were indeed botched -
probably due to a mixture of DOS and Linux style line breaks, which made the
string retrieval fail - affecting various places in the extension. This change
will be rolled into the next official version.

Please try 1.8.5pre9  (pre8 did not fix the issue).

If you see any English strings these might be new options that have not been
translated yet.
