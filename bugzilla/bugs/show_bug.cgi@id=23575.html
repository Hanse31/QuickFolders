Description:   Opened: 2010-12-29 15:16 
After dragging a few messages to subfolders of a QF folder, all dragging of
messages in Thunderbird fails.  I.e., I click on a message in the thread pane
and attempt to drag it to a folder in the account pane (or to a QF folder). 
Instead of the cursor turning into a hand icon dragging a message icon, the
cursor remains an arrow, and nothing happens when it reaches a folder. 
Quitting and restarting Thunderbird restores normal operation.

OS:  Fedora 13 linux, kernel 2.6.34.7-66.fc13.x86_64 #1 SMP.
Thunderbird: Version 3.1.7
QuickFolders: Version 2.2

Reproducibility:  The bug doesn't show up immediately.  I can drag several
messages to QF subfolders and everything is fine, but sooner or later (usually
sooner) the mouse "loses" the message it is dragging, and from then on,
dragging doesn't work at all.

This sounds like a drag and drop operation not being properly finished.
unfortunately, debugging drag and drop operations is one of the most difficult
tasks (as usually the mouse is in captured mode during debug operation which
makes it very difficult to operate the debugger). The problem is further
exacerbated by the fact that the code to teach QF to accept dragged emails on
the popup menus (and even the parent nodes) is already fairly complex, which is
in the nature of the beast.


> I just now noticed that the keyboard froze up on all
> of my windows (including terminal windows) and remained frozen until I killed
> Thunderbird, whereupon everything started working normally again.
> 
> I'm guessing that QF is not managing system resources properly.
> 
I would not even be sure about this statement... drag and drop is a complex
operation and it might well be implemented in a different way depending on the
operating system. There is some special code that deals with the collapsing of
parent menu items on Linux installations but it is only relevant to Thunderbird
versions before 3. So I remember that the drag & drop implementation in Mozilla
under Linux was "flaky". If you can come up with some error messages from the
debug console this might be helpful in reproducing and resolving this problem,
but I would not hold my breath for it - there were numerous weeks put into the
design of the drag and drop code and the folder popups and to fix something
like this is going to take a considerable amount of time.

It might also make it necessary to build some test versions that add special
debug outputs for further testing, if you are willing to get to the bottom of
this then I would attach them to this bug.
