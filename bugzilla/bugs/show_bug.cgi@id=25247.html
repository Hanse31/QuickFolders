Description:   Opened: 2012-11-30 22:32 
I am running Thunderbird 16.0.2 and ubuntu 12.04, SmartTemplate4  0.9.2.

I place this date string as the first line of an email in SM4 Preferences:
"<em>%B% %e%, %Y%</em>"
and on a reply or Forward I got today's date at the top of my page, as expected
but my new message start is dated  December 31, 1969.

I began by editing my Reply tab, then copied the code to clipboard and pasted
into the Write and Forward tabs.

Here are my Preference settings:

Write tab:
Checked all three boxes, to use the following template, to use HTML, to use
<br>.
Entered the code as a copy-paste from the Reply tab.
<em>%B% %e%, %Y%</em>

Reply tab:
Checked all four boxes to use this template, to disable the default header, to
use HTML, to use <br>.

Entered a Reply header in the top box to wit:

In reply to %from(name)%, on %X:=sent% %a%, %b% %e%, %Y%, at %l%:%M%%p%:

Entered the date code from scratch in the bottom box.
<em>%B% %e%, %Y%</em>

Forward tab:
Checked all boxes except to disable the default header.

Top box is empty, bottom box contains copy-paste of the date code.
<em>%B% %e%, %Y%</em>

My Reply header displays correct information.

There is one more item I did not communicate to Axel when I reported the above
information:

I wanted to use the %tz_name% on the end of my reply header on the reply tab,
but it only displayed one letter, "M" instead of "MST" for United States
Mountain Standard Time (-0700 UTC).
