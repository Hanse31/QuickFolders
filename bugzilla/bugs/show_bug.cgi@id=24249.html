Description:   Opened: 2011-07-30 01:27 
Hi, first like to say what a fantastic extension you have created! 

Quickfolders version: 2.7.1
Thunderbird version: 5.0

When Thunderbird upgraded from 4 to 5.0 it was instantly behaving laggy. By
going through all the Extensions I was able to determing that Quickfolders was
causing the general slow and lag when using all aspects of Thunderbird.
Everything from switching between tabs, opening and email and even a menu item.
All actions in Thunderbird now have a noticeable lag.

The lag doesn't seem to get exponentially worse with time. It is laggy from the
start of application to the finish. Once disabled and rebooted, Thunderbird
returns to normal application speed.

Task Manager reports memory usage the same with the extension active or
disabled. Both started at 90mb on launch. I did perhaps see higher peak usage
of 200mb with the Extension on after an amount of time.

All accounts are IMAP. I have around 10 accounts in this install.

Happy to record a screencast to show you.

Thanks

Thanks for this;

OK, it is hard to say from just this, but its obviously an IMAP specific issue.
Can you try and check your Synch&Storage settings and make sure the options
"Keep messages for this account on this computer"(*) and "SYnchronize messages
locally" are both checked.

(*) Also in the advanced options make sure your folders are selected for
offline use

this should avoid round tripping to the server to find out folder sizes. You
could also try to disable the options "Display the Number of Unread Messages"
and "Also display totals of Subfolders" to see if this improves the lag. I have
a fairly slow broadband connection (but a fast computer) and could not notice
any perceivable difference. You should note that a click on a folder tab will
always set the opened folder in the tree first and this will then set the event
to update quickfolders, so there can be a very short waiting time (typically
less than 0.5 secs). Also the message counts are updated via a timer, so these
do not update immediately. This is done to avoid overloading the Thunderbird
engine and leads to an overall faster responsiveness. On IMAP accounts, numbers
of unread messages can increase and spawn many folder load events, through the
timer I am able to bundle them into one quickfolders update.

It might also be significant what OS you are using, there were some unnecessary
stack messages created by TB on Macs(**) (due to some untidy programming on
Mozilla's side). For the moment, you can contact me directly (via email) to
resolve the issue; maybe we can set up a screen sharing session on
http://join.me at some stage.

(**) I do not have a test system for Mac, but I do have a Linux VM.
