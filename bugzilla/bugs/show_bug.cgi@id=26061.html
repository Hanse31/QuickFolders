Sorry but these error messages are not useful. Like I said, please install
console2 which I attached in the first attachment and set it up the way I told
you.

Make sure to diable "Content", we are only interested in "Chrome" and "JS"
messages. Also,  screenshots are unfriendly for logging - you can use the
right-click menu to
- Select All
- Copy All messages

and then simply copy the text into the next bug reply. 

Make sure to clear the messages (empty list) first, then right-click any folder
in the folder tree (left hand side) and "open in new window". This will open a
new Thunderbird Window / Process and we will get a glimpse at the errors that
would also happen at start-up [unfortunately logging the errors on the "real"
Thunderbird start often times does not work, that's why we use the "open in new
window" workaround.

You should get one log after opening the window. You can also add  more error
detail by opening ST4 settings > advanced functions and enabling [x] Debug
mode. Additional messages are logged when you right-click that check box and
toggle the settings:

extensions.smartTemplate4.debug.functions = true

If this doesn't come up with any warnings or errors, check Thunderbird Options
> Advanced > General [Config Editor...]

in search, enter: 

extensions.smartTemplate4.showStatusIcon

is it shown more than one time?
