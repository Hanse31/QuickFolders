Description:   Opened: 2016-03-20 07:10 
Dear Axel, 

First of all, many thanks for creating quickFilters. It has saved me a huge
amount of time. 

To make it even quicker, would you consider adding a basic macro function?
Here's how I think this could work:

At the moment, I use quickFilters mainly for the "Mailing List” feature.
Whenever I receive, say, a sales e-mail from a new source, I add it to a filter
called “Sales", to direct any future e-mails from the same source into a
folder called "Sales" automatically.

This works really well, but at the moment it requires me to click through
several dialog boxes to achieve quite a simple outcome. All I want to do is add
the sender's e-mail address to a long list of addresses in the relevant filter.
But to do this, I have to:

1. Drag the e-mail to the relevant folder (with the filter assistant toggle
on).
2. Click on the relevant filter after quickFilters detects that I want to carry
out a merge.
3. Click on “Edit filter” (assuming the correct template is highlighted
above).
4. Click OK.

What I'd prefer to do is create a macro called, say, "Add sender address of
selected message to Sales filter" and then make this macro available as a
toolbar icon. This way, whenever I receive a sales e-mail from new source in
future, I could simply highlight the message and click on the relevant button
to add that source to the filter and move on.

I hope this is of interest and look forward to hearing what you think. And
thanks again for creating such a useful extension.

To shortcut the current steps as much as possible, you can select the options
on the "Actions" tab:

[x] Automatically Select Merge
[x] Skip to Template step

I think this works as long as there is only one filter to merge with. Is this
the case here or do you have multiple filters that send mail to that folder?


You can also uncheck the option "Display Filter list after filter creation"

AS regards having a "quickMerge" button that is _specific_ to one particular
folder, I would have to store it somewhere (at least the folder URL and maybe
the filter name and possibly which template to use for adding the new rule - in
your case it would be based on sender) and then make it possible to create
multiple button instances - unless we had only one button and a dropdown of
mailing lists, which might be less work for me. So instead of having one button
per "macro" have a list of merge candidates.

In your case the dropdown would have a menu item
"Merge with Sales Filter"
And clicking it would
1) extend the filter with the sender
2) run the filter to automatically move all emails from this sender to your
"Sales" folder. Possibly it would show a notification with a summary 

(x emails moved to Sales folder)
