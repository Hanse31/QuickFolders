Description:   Opened: 2012-01-19 06:20 
I hope I didn't miss this already being covered in my searches....

I currently use the "favorite folders" list, which includes a number of folders
where filters move messages to. In that list the folder name turns blue to let
me know mail has been dumped there that I have not seen yet. It's easy to see
where new mail is waiting among those favorite folders.

If I use "all folders" those particular folders aren't always visible (i.e.,
they're subfolders in unexpanded folder trees).  So my normal mode is to keep
"favorite folders" displayed, and it's easy to see my new mail arrive in all of
the various folders as it comes in.

Now I've added QuickFolders. I've created a QuickFolders category I call
"Inboxes", which gives me a set of tabs for all of those folders described
above.  Works just fine, EXCEPT that the text on the tabs don't turn blue like
they do in the folder list. I end up having to always watch the folder the list
for the new mail arrivals, which kinda defeats the whole purpose of having the
"Inboxes" listed across the QuickFolders tab bar.  And I have to always leave
"favorite folders" in the folder list to see those blue highlights over there.
This, too, interfears with my use of QuickFolders to include other non-favorite
folders, which automatically switches the folder list back to "all folders"
when I select a non-favorite folder with QuickFolders.

My request is to provide the "highlight tab" function to match what happens to
the tab text in the folder list.  If the folder in the folder list would be
highlighted to indicate new mail has arrived there, the corresponding
QuickFolders tab would also show some kind of highlight.  Whatever method of
highlighting is easiest to do for you is fine with me, just give me a way to
look at my QuickFolders tab bar and see what folders listed there have new mail
in them.  PLEASE PLEASE PLEASE.  Thank you...

------- Comment #2 From David L 2012-01-19 07:10:14 ------- 
Found the patch I mentioned. Added following to userChrome:

/* Display highlighted text on folders with new messages */
treechildren::-moz-tree-cell-text(folderNameCol, biffState-NewMail,
isServer-true),
treechildren::-moz-tree-cell-text(folderNameCol, newMessages-true),
treechildren::-moz-tree-cell-text(folderNameCol, specialFolder-Inbox,
newMessages-true) {
  color: -moz-hyperlinktext !important;
}
treechildren::-moz-tree-cell-text(folderNameCol, biffState-NewMail,
isServer-true, selected),
treechildren::-moz-tree-cell-text(folderNameCol, newMessages-true, selected),
treechildren::-moz-tree-cell-text(folderNameCol, specialFolder-Inbox,
newMessages-true, selected) {
  color: -moz-CellHighlightText !important;
}
treechildren::-moz-tree-cell-text(folderNameCol, biffState-NewMail,
isServer-true, selected, focus),
treechildren::-moz-tree-cell-text(folderNameCol, newMessages-true, selected,
focus),
treechildren::-moz-tree-cell-text(folderNameCol, specialFolder-Inbox,
newMessages-true, selected, focus) {
  color: HighlightText !important;
}

If there's anyway to use logic like that with QuickFolders, please advise....

By default, any Tabs that contain unread emails, their text turns bold. We are
not currently using colors to highlight unread emails. Maybe you have
configured your system in a way that the font for the folders is always bold?

I would recommend turning on the following options on QuickFolders:
[x] Display Tabs for folders with unread messages in bold
[x] Display the number of unread messages on the folder button
[x] Also display totals of Subfolders

that should give you what you want, out of the box, with the default theme. I
have also used Mostlyt Crystal until recently, and I am using most of the
wellknown themes in rotation, haven't found one that this is incompatible with
yet.

Maybe you could provide a screenshot as an attachment?
