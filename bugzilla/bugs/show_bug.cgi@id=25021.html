Nope, that is not related. That one had to do with a change in the xul
structure. The new Mac bug is very likely a gecko issue, and could be related
to hardware acceleration or other low level changes that only affect that
platform. it is unusual in that QuickFolders does not actually do any painting
within the messages list itself. I won't be able to even attempt fixing this
without a (hardware) test platform and a debugger.

e.g.

        -moz-box-flex: 0 !important;
        -moz-box !important;
        background-color: yellow !important;
}
Another thing to be really sure would be to check with DOM inspector. It is
able to tell you the computed CSS styles of everything within Thunderbird. It
can also draw an outline around the object, so this might be helpful in your
case as well.

I was also thinking, maybe it is not the toolbar but the container of the tabs
(it is called QuickFolders-FoldersBox). try adding a rule:

#QuickFolders-FoldersBox {
  overflow-y: hidden;
}
just to see what this does? The disadvantage of this rule is that you will
loose the ability to have more than one row of tabs.
