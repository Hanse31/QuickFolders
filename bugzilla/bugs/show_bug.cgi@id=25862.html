Description:   Opened: 2014-09-30 12:45 
* Totally on the side, but something I really miss.
   Ability to link a mail address to an account.

I have quite a few accounts and it's so easy to forget to set the right account
when composing new mail. Somehow after the 'To' is filled out the account is
changed accordingly.

Preferably this would be set in the address book somehow

You can do it !  ;-)
------- Comment #1 From Axel Grude 2014-09-30 13:18:12 ------- 
I definitely agree with this one. My workflow in a lot of cases is: 

1. go to a folder that is dedicated to an individual or mailing list
2. write a new email to... the main person or list

so it would absolutely make sense to have a "default contact" with any folder
(and / or quickFolder tab). And to fill in that mail address when creating a
new mail (while I am viewing this folder). Another additional step might be to
also associate a Stationery Template for the person, which could be tied to the
folder as well.

I always thought I was the only one with that use case, but it totally makes
sense to me that you would find it useful as well.

Reading your request again - it somehow looks like I misunderstood your
request. What you want is the "To" Address to select your "From" address, 
which is not really something that falls under the category "folders" at all.
It is true that the location of a folder does determine which identity the
message is composed from ; this has nothing to do with the "To" address; you
can manage this by simply moving the folder into the correct mailbox. Can you
clarify that this is what you meant? (Select To in Composer will set From)

What I thought you meant was that _the folder_ would determine the to address
and this is something which is much more in scope, but I may have to raise a
fresh bug on this.
