Description:   Opened: 2014-05-25 12:56 
I'm not sure if this is the same issue as 25777 - It might be, in which case
feel free to consolidate.

QF3.14.1, TB 24.5.0, Windows 7 (Ultimate)

QuickFolders toolbar has started misbehaving as described below.  (+) indicates
expected behavior, (-) indicates problems.

I have two Inboxes (different accounts) and three other folders on the QF
toolbar.

If I open a mail message from an Inbox, I can click on the QF Inbox & navigate
from the message tab back to the inbox. (+)

However, if I have a folder listing open (I don't use the mail preview pane),
and click on the QF folder link, the link is underlined (indicating selection),
but the folder listing pane does not change. (-)

If I click on one of the QF-pinned folders on the tree view, the listing pane
changes as expected.  And the QF link is underlined. (+)

If I click on a non-pinned folder, the QF link underline is removed. (+)

If I open a message from the Inbox, switch to the inbox tab, select a different
folder, and return to the message - clicking on the Inbox link has no effect.
(-)

Finally, if I open a message from one of the other pinned folders, I can use
its QF link to return from the message to the containing folder (+), but NOT to
the Inbox (-).

So it seems to me that the issue is that changing folders by clicking on a QF
pinned link has broken.

The failing sequences do not make any entries in the console log or activity
log.

I have not installed the pre-release attached to 25777- but can if that would
help.

This is 100% reproducible.  The misbehavior started after TB was restarted
following the update to 24.5.0.

Thanks for your help!

to be honest I had seen something like this before I am just currently looking
at the change set to determine what exactly might have caused it. One problem I
saw very recently in Thunderbird (Beta) was that the tabContainer did not have
its selectedIndex set (it should be zero if you open a new tb window), so I
guarded against that. There were also some improvements in my function

QuickFolders_MySelectFolder()

which takes care of most of the card work when the current folder gets changed
(I know it sounds trivial, but it is actually a really difficult function, as
there are numerous things that can go wrong across the various host programs
SeaMonkey, Thunderbird and Postbox). Thunderbird's treeview code has a new
parameter for its function TreeView.selectFolder that forces selection, so this
might have helped.

There is a new setting in the Debug log settings called 

extensions.quickfolders.debug.folders.select

which can now be employed for detailed error logging when selecting folders,
but since you cannot reproduce the error anymore it is probably pointless for
now. IF it happens again, you should try and activate the above setting via QF
options > advanced > [x]debug > right-click on the debug checkbox.

Apart from that, I am happy to flag this bug as resolved FIXED.

Note: There have been numerous other changes in this prerelease, and some of
the new (premium) functions such as Setting Folder tree icons or the quickJump
feature will likely be available only for registered users once 3.15 goes into
release; but they are free as part of any beta / prerelease version.
