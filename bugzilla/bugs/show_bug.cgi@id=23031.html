Description:   Opened: 2010-07-19 14:36 
I recently bought my first wide-aspect-ratio monitor, and after using
Thunderbird and QuickFolders on it for a while, had this idea.

   I have lots of QuickFolders tabs (>25, because your software is so good!),
and currently I have to write very short names for them in order to make them
fit in the QuickFolders toolbar across the top of the Thunderbird panel.  It
occurred to me that if the user had an option by which the toolbar is arrayed
vertically along the side instead of horizontally along the top, with the text
of each tab still horizontal, it would make better use of the screen width to
permit longer tab names, and the number of tabs would be limited by the screen
height, with each tab occupying only one "line" of height.  (If their font size
were also adjustable, the user could squeeze in even more.)

   I don't know whether it's even possible to dock a Thunderbird toolbar along
the side, and I'll leave prioritization decisions to you, of course.

   I also just recently discovered that the QuickFolders tabs drop a cascading
menu of subfolders - has that always been there, or is it a new feature?  I've
been using QuickFolders since December, but I had not know about this feature
until this week.  It's really nice and helps to cut down the number of
QuickFolders tabs.

------- Comment #3 From Axel Grude 2010-07-20 05:35:48 ------- 
Hi Charlie,

probably not [easily]; as it is a toolbar and these are generally horizontal. 
You could however just drag the folders into the Favorite Folders View this
would give you roughly the same functionality, just vertically.

Please note that all newer versions of QuickFolders support wrapping the tabs
to multiple lines, so there is no restriction to the number of tabs you can
have in one screen. Also there is the categories feature which breaks the tabs
down into smaller more managable groups.

------------------

My thought about the toolbar layout was that it is similar to the Personal
Toolbar Folder in Firefox / Netscape, which was one of my most important
reasons to change from Internet Explorer (which only had a vertical bookmarks
list) to (then) Netscape 4.0. As opposed to a long list that you scroll down
(and which you already have with the All Folders toolbar),  it maps out your
starting points side-ways, and then adds folder submenus for a quick,
many-dimensional lookup.

So my first point here is - there is never any scrolling involved - very
important when you are doing "drag and drop" for  moving email.

Second point,  initially dragging vertically up then down (to move mail) to hit
a target is surprisingly much easier than side-ways and then trying to hit the
correct folder, in my experience. I have often mis-moved items in the Windows
Explorer File tree and even Outlook because I did not hit the right target or
the list scrolled (which in Internet Explorers Favorites is made even more
complicated by expanding / contracting nodes). The longer your folder list
(mine would be around 10 screen pages high, when fully expanded) the more
cumbersome any drag operations become.

IMHO, in a vertical scroll box - the same amount of targets would become a real
usability nightmare

The last point is that the Folder bar is already a vertical representation of
the folders, so adding this view mode into QuickFolders feels like an
unnecessary duplication. And I must say, I really like tabs! Which brings me to
another point, when you open multiple tabs it seems more logical to change
their locations with the tabs above:

In an example (see the 2 attachment pictures 'Using a Folder Category...'),  I
have 2 Thunderbird tabs open, one with a Mozilla Folder, the second one with
Folder in my "Friends" Category: When I change the tab using the mouse,
QuickFOlders will automatically adapt and show the relevant "sibling" Folders. 

So this is like I can alternately work in 2 different Categories (e.g. Work /
Private) without having to think much about Navigation.

Having said all this I don't think I will put the necessary work in to get away
from the tabs metaphor - after all the Extension is also called "Folder Tabs".
Something like this should probably go into its own Extension (and there might
even be an Extension that does exactly this).

One more thing - you can adjust the font size of the QuickFolders tabs on the
second tab of the options window.

The subfolders menu has been there at least since version 0.9. I think I added
drag & drop support shortly after that.
The recursive sub-subfolders can also be displayed since version 1.6. 
In 1.9.3 I simplified this by taking out duplications.

The full Version history is available here:
http://quickfolders.mozdev.org/version.html

Thanks for your suggestion.

Axel
