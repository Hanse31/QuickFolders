Description:   Opened: 2011-06-30 16:54 
TB v3.1.11
QF v2.7
Windows 7 Home Premium

I have been working with about 30 QF bookmark tabs in a variety of colors for a
long time.  I'm very happy with QF, and have become dependent on it.  It was
automatically updated to v2.7 in the past two weeks or so.  2.7 was working
fine, but tonight I accidentally tried to create a new tab for a folder that
already had a tab.  I got a message "Folder is already bookmarked."  I clicked
OK, and immediately most of my tabs disappeared from the QF bar.  Only "Recent"
plus 4 others remain.  The remaining ones were not contiguous before the error.

I tried stopping and re-starting TB, disabling and re-enabling QF, and complete
reboot of Windows.  No change.  I hesitate to uninstall QF if there's a chance
my bookmarks information can be saved, since I have so many.

I cannot reproduce the behavior because I cannot return to the starting state
with all my tabs.  If I deliberately try again to create a duplicate bookmark,
the error message appears, but there is no further change to the QF toolbar.

I turned on the Error Console (Console-2 v0.7), but nothing appears there.

------- Comment #2 From Axel Grude 2011-07-01 00:27:28 ------- 
Hi Charlie, *before* you try to fix this problem, can you please send me the
folder string via email? Go to QF Options / general and click the [Folders?]
button, then paste it and send in an email.

My initial guess is that either a category was defined (unlikely as you would
probably know this) or one or more of your folder names contain double
quotation marks. After you have copied the folder string try the following:

In the QuickFolders main context menu (click on the cog wheel or right click an
empty area on the QF toolbar), select "find orphaned tabs" and follow the
instructions on screen; this will remove any tabs that point to invalid /
unfindable folders; this might already resolve the problem.

------- Comment #4 From Charlie 2011-07-01 07:26:04 ------- 
(In reply to comment #3)

OOPS - Mea culpa!  I just refreshed my memory about Categories, and now it all
makes sense.  Yes, I had defined two categories a long time ago, but have
always left the button set at "Display All" and completely forgotten about it,
using colors instead to differentiate groups of folder tabs.

Now I see that I must have accidentally clicked that button (it's very close to
the sprocket wheel icon for Options) and switched to a limited category, which
changed automatically if I manually select a different folder.

Since I don't actually use the categories, I am deleting them to prevent future
confusion. I see that this also deletes the Category selection button, which
gives me more room for my many tabs on the toolbar!
------- Comment #5 From Axel Grude 2011-07-01 09:47:04 ------- 
deleting categories resolved the problem. Note that if there is a drop down
displayed on the left hand side of the QuickFolders bar, then this means some
sort of filtering is applied (unless the entry "Display All") is selected).

Just in case anyone has this problem later, I want to note that deleting
categories does solve this problem, but is NOT REQUIRED to solve it.  You can
keep your categories, but just don't ignore the setting of the categories
drop-down.
