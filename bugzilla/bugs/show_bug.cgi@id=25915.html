Description:   Opened: 2014-12-11 23:29 
I'm running SmartTemplate4 0.9.6 on Thunderbird 24.6.0
I started seeing this behavior recently, but I'm not sure if it's due to the
latest update of SmartTemplate4

Some (but not all) long headers are truncated when replying to a message. For
example, if I receive a message with a long subject, when I reply to that
message the quoted subject is truncated. Other headers (date, etc.) are not
truncated, maybe because they are much shorter.
The truncation point isn't the same for all messages, but seems to be around 60
characters or so.
However, I'm only seeing this with about 10% of the messages I receive. The
other 90% are not truncated in replies, even when they have very long subject
lines.
One of the few things that all affected emails have in common is that they were
written in Spanish (my native language). That seems unlikely to be the cause,
but it's the only thing in common that I could find.
The truncation is perfectly reproducible, all affected messages are always
truncated at exactly the same point.

I'm using this template on SmartTemplate4 for replies:

<br /><br />
<hr style="height: 1px;border: none solid
#000;color:#000;background-color:#000;" />
<b>De:</b> %from%
<b>Fecha:</b> %date%
<b>Para:</b> %to(name)%[[
<b>Cc:</b> %cc(name)%]]
<b>Asunto:</b> %subject%
<br />


I have a few emails that exhibit this behavior, but I'd rather not post them
here for privacy reasons. Please let me know if you have an email address where
I can send them as attachments.

Let me know if there's any additional information I can give that may be useful
for debugging this or if I can help to solve it in any other way.

Thanks in advance.
