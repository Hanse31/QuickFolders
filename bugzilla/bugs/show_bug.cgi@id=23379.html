Description:   Opened: 2010-10-16 06:28 
It would be nice to be able to access Thunderbird's "Recent Folders" list
through QuickFolders, or vice versa.  I searched Bugzilla and the forums for
"recent" and was unable to find anything for QuickFolders on this topic.

I can think of two good ways to do this, and you might think of more.  These
two are not quite the same as each other, and complement each other.  They
represent the flow of information in two directions between Thunderbird and
QuickFolders:

(1) When a folder is accessed via a QuickFolders tab, add it to Thunderbird's
Recent Folders list.  This would make it accessible through native TB commands
such as right-click on message -> Move To -> Recent.   I don't know whether
this is possible through code.

(2) Make an option in QuickFolders to have a "Recent" tab in the QuickFolders
toolbar, with a dropdown of subfolders like the other tabs.  I have seen
fleeting references in Internet postings to this being an existing feature, but
I can't figure out how to actually create it.  To be most useful, the dropdown
list should be dynamic, i.e., automatically updated when Thunderbird changes
the list.

P.S.  I'm using Windows 7, which does not appear in the Bugzilla dropdown for
OS.

Thanks, as always, for your consideration.

------- Comment #2 From Charlie 2010-10-16 22:23:10 ------- 
I did know about the Recent Folders list that is available as one of the
choices in the left-hand Folders Tree pane, and that is indeed ROUGHLY the list
I had in mind. It's true that clicking a QF tab to move to a folder does add
that folder to the Recent list in that pane, but in my hands, dragging a
message to a QF tab, which I do very frequently to move my messages into
folders, does NOT add the folder to that list unless I also then refresh the
list by switching the left pane to a different list, then back to the Recent
list.  (I am now in TB v3.1.4, but I believe this has been true for at least
several older versions as well.)

However, I find it confusing to switch lists in the left pane because I almost
always want to see all folders there, and sometimes forget to check which list
I'm looking at.  Moreover, I work with several e-mail accounts (addresses) in
the same TB window, and the list of Recent Folders in the left pane appends the
account names to each folder, making the names too wide to fully display in the
pane, as I have it configured as a fairly narrow column.

There is another not-quite-identical "Recent Folders" list that I actually use
much more often and doesn't require taking the mouse to the far left.  If you
right-click on a message in the message list, or anywhere in the body while
reading a message, the context menu contains three items that involve a
"Recent" list, namely
a) Move To -> Recent
b) Copy To -> Recent
c) Move/Copy to "Last Destination Folder Name" Again (the name in the quotes is
presumably at the top of a list of Recent Folders).
I would like it if dragging a message to a QF tab automatically pushed that
destination folder onto the lists used by these context commands.  That's what
I meant by my original suggestion (1).  At the moment, in my hands, it does
not.

My suggestion (2), as you realized, largely duplicated the "left-pane Recent"
functionality.  I really only mentioned it because I saw references to it and
wondered if I missed an existing feature, and how to create it on the QF
toolbar, where its tab could be renamed, positioned, colored, and always
unmistakably visible.  That would be "nice to have," but my suggestion (1) is
more important to my work.

BTW, there are many Internet postings complaining that TB's internal "Recent
Folders" list(s) are not maintained accurately.  For examples, see the output
of Googling the phrase below, with the quotes as shown:

thunderbird "recent folders" list

I do not expect you to solve that problem, which I think is mostly a difference
of opinion about what the algorithm should be concerning what actions cause a
folder to be pushed onto the list.
