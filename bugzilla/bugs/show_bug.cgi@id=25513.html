------- Comment #5 From Axel Grude 2014-07-25 07:58:53 ------- 
I "might" be adding it to the main options, but the Dialog is already very very
complicated. Too many options make it hard to find the more important things.

If you are using the feature regularly, would you also neeed to toggle it off
regularly? This question is important as it shows that the use case might not
be as clear cut as you have put it initially ("always mark all mails that I
file away as read")

If your use case changes (e.g. you might have 2 types of mails, one that you
always want to mark as read and another that you want to  alwaysfile away for
reading later: in this case it would be interesting to learn about the
differences and find out what can be learned from a UI perspective. Or maybe
you have 3 kinds of mails, 1 and 2 as described above and 3 a group of mails
where is it not so easily determined - you might want to mark 65% as read and
leave 35% as unread. These types of classifications are extremely important as
they can require very different UI experiences) and you notice that you need to
toggle this option back and forth regularly, I would certainly like to know,
and learn about the decisions behind this.
