------- Comment #10 From Axel Grude 2019-03-12 09:08:47 ------- 
Created an attachment (id=8610) [details]
3.7.1 prerelease 34

Added code guarding against the exception "searchTermList is null" (shouldn't
happen, but there you go).
if no list-id is found (or the custom header for list id is not defined) then
it will use the message.author field for the "from" condition. It will also add
any ccs of the message to the cc condition.

Once list-id is defined for any other filter there should now be no warnings at
all using the "mailing list" template.

- note there is a new warning when you "create filters from message" on an
email that still sits in the inbox and has no labels attached. This an
additional feature unrelated to this change.
