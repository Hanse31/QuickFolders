Description:   Opened: 2016-07-23 03:50 
create a Hungarian (standard) keyboard layout for Zombiekeys

I am assuming the term "standard" which is based on  a QWERTZ keyboard layout
is the most common one used in Hungary so I am going to base the layout on
this. There apopears to also be a variant Hungarian Hungarian
(101/qwertz/comma/dead keys) which has the Pipe | character at the bottom left,
not too sure how common this one is so there may have to be a discussion on
which one exactly has the most merit.

According to the anonymous usage statistics there are only 1-2 users of the

magyar (hu)

locale, so it is a little difficult to decide for the correct one. It may well
be that the usage would increase once the locale is added.
------- Comment #1 From Axel Grude 2016-07-23 04:00:34 ------- 
From https://en.wikipedia.org/wiki/QWERTZ#Hungary

Note that on some keyboards "ű" is found to the right of the right Shift key,
or the right of the "ó" key with a smaller backspace key and larger Enter key.
Also, many Hungarians who were not exposed to typewriters, but started using
computers when Hungarian keyboards were uncommon or unavailable, still prefer
the layout with the y/z keys in their QWERTY locations because that is what
they are used to. The same applies to Hungarians who frequently switch between
a QWERTY and a Hungarian layout (for example because they write in several
different languages).

I had some trouble at first recognizing the many different diacritics that are
mapped already on an Hungarian keyboard, my general experience is that there is
almost a complete mapping in the form of ALT GR symbols, so I decided to
largely use these keys for the mapping instead.

For instance accent grave and acute are now mapped to the [7] and [9] keys as
they depict these.

Note that I am using CTRL+SHIFT for the first 4 number keys as the  shortcuts
CTLRL+1 .. CTRL+4 are actually use for the tabs in Thunderbird. The SHIFT key
is not necessary for the many extra diacritic keys specific to Hungarian
keyboard which are unlikely to be used for existing shortcuts.
