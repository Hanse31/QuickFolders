two screenshots of the e-mail in UltraEdit

I'd to change a working filter for immobilienscout24 where in the past the
subject was e.g. "Wohnung Miete (Umkreissuche)" and I knew that this search
belongs to a search for an appartement in Nuremberg. But because of the latest
change I could no longer differ the different searches.

Therefore I changed the filter from Subject to Body (Inhalt) and replaced the
content to search for to "Nürnberg, Provisionsfrei".

But since then the filter no longer worked. I made some test filters for other
e-mails and they worked fine with the content search inside the body. The I
took a look at the source code of the e-mail and it seems, now I'm speculating,
that there is a problem with the German Umlaut in "Nürnberg". The charset of
the e-mail is ISO-8859-15, but they don't use in this case "&uuml;". Though, if
the message is not searched after converting it into this charset, the Umlauts
won't be recognized. If the Umlauts are correct translated to "&auml; etc. it
works fine as some tests showed.

Of course I already wrote a message to ImmobilienScout24 and told them that
they should use UTF-8 charset if doing so but I don't know if they are smart
enough to understand this problem or are willing to change the charset.

The upper part of the screenshot, taken in UltraEdit, shows encoding set to
ISO-8859-15 and the lower part set to UTF-8.

Hinting greetings

Reiner
