------- Comment #2 From Axel Grude 2012-07-03 06:59:21 ------- 
The current design removes anything between { } if there isn't a variable
contained (like %cc%); if the variable points to an empty value (e.g. no cc
given) then the contents are also removed.

I guess the original designer of ST4 did not think of people wanting to style
their signatures using css rules. 

I am currently discussing a fix of this in future versions by replacing the
currently required {} characters for optional smartTemplate variables with
other (double) characters, such as [[ ]]. 

This will take a little while as it breaks compatibility with existing
templates, so a conversion routine must also be written.

------- Comment #4 From Axel Grude 2012-07-05 03:10:17 ------- 
We have developed a solution which does what I proposed:

- the new method to mark optional lines in the template is now with double
square brackets:
[[  ..%variable% line here..  ]]

- ST4 will automatically import the existing templates to work (by replacing
all { } characters with [[  ]]

- You can now add CSS styles and/or use { and } in your template and signature
without triggering the bug.

I will post a patched version shortly - please note that the conversion to the
new format is "one way" - the old templates are backed up in the profile, so if
you should revert to an older version of ST4, they will also revert to their
old versions (at time of upgrade). I am also working on a User Interface that
communicates these changes during upgrade.
