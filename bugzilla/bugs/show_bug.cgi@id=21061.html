Description:   Opened: 2009-06-12 20:40 
Running Thunderbird Portable version 2.0.0.21 (20090302) and Quickfolders
1.0.3.Also running Lightning 0.9, Adblock Plus 1.0.2, Bork 1.7, Buttons
0.5.3.2, Contacts Sidebar 0.7.1, NotTo 0.4, Select Addressbook Text 1.0,
TrackPackage 2.4.   Using default Thunderbird theme.  Using Thunderbird master
password.

At Thunderbird startup, the bookmark bar is present however the bookmarks are
invisible.  Righ-clicking on the bar and selecting Options, Ok (with no
changes) - or - clicking on the bar and selecting "change the order...." (with
no changes) and Ok makes them appear.

When closing and reopening Thunderbird after the change, the first set of error
log entries appear and the bookmarks are invisible again.

Tried reinstalling, deleting installed quickfolders app files, resetting all
quickfolders entries in about:config.

The bookmarks showed up properly the first 3-4 times, then suddenly became
invisible.  

This is by far the most useful extension ever - very curious and I hope it's

------- Comment #1 From Axel Grude 2009-06-13 03:40:14 ------- 
Just got a report from another user reporting the same. Usually tabs not being
shown is a result of the function updateFolders(true) not being called, but
that's the case with yours - are you using Categories? If not, try to add tab
to a Category and select this then restart TB to find out if the problem
persists.

You can also send me your folder string (private message, because it will
contain email addresses) mayber there is a clue in there somewhere.

One theory is that if no Category is selected something goes wrong - will test
it this weekend...
------- Comment #2 From Axel Grude 2009-06-13 05:43:26 ------- 
I have tested this by deleting all Categories  and also tried manually setting
various values for
extensions.quickfolders.lastSelectedCategory
(about:config)
but I could not reproduce it so far. There is a function shouldDisplayFolder( )
in Quickfolders-interface.js and I guess this is where the problem is. If
you're not using categories the folder string should display values like this:
{"uri":"mailbox://realraven@mail.netscape.net/Studio/M-Audio",
"name":"",
"category":null}

a category of 'null' should always display the folder...
------- Comment #3 From Axel Grude 2009-06-13 10:05:19 ------- 
I have managed to reproduce this by lowering the "Init Delay" on the first
options page. delays of 10ms or less produce a completely empty Quickfolders
bar (no message "drag folders here" or any Quickfolders label - is this what
you are getting?). 

The default here is 1000ms, could you try whether setting this to 1500 ms makes
the bug go away?
------- Comment #4 From steve 2009-06-13 12:57:40 -------
