Description:   Opened: 2018-12-01 10:32 
Thunderbird 60.3.2
Windows 10 latest.
QF Pro, paid. Version of 24 Nov 2018

I've used several of the icons recommended in QF.
In each case, the selected icon displays correctly in the folder tree.
Upon restarting Thunderbird, QF "knows" that an alternate icon does exist for
Folder A (for example), because the toolbar has the "remove customized icon"
icon.
But the displayed icon for Folder A has reverted to the default icon.

Troubleshooting thus far:
I've experimented with various settings for QF, including foldertree icon
settings.
I've placed the icons in various folders, and Setting shows that QF has the
correct path.
The correct path perists between Thunderbird sessions.
I've ruled out interactions with 
Dorando keyconfig,
Nostalgy,
Theme & font size changer.
