------- Comment #4 From MarkyMarkDE 2012-11-11 05:29:34 ------- 
(In reply to comment #3)
basicly: thunderbird pasts below your message a "header line" like 
"on date author wrote:" or the table on forwarding message with the full mail
details.
We search this elements and replace it with the user definied template this is
the design of the Add-on.
On a new Message you don't have such elements in case that is an complete blank
mail.

We preffer in the future to use the signature Settings inside Thunderbird
Account settings as to set the %sig% and %sig(2)% variables in your template.
-> you see self why!

You can style with css to formate your signature same as your template.
Make an id and give them the same font-family, size and color as your template
e.g.

<div id="mySig" style="font-family: tahoma, sans-serif; color: #000000;
font-size: 12pt;">
  here comes now your signature phrase
</div>

This you do with all of your signatures (set inside account settings of
thunderbird per each identity) which you use inside Thunderbird and all should
be worked well.

------- Comment #12 From Axel Grude 2012-11-11 12:41:54 -------

> with the present (french) version of SmartTemplate4 for answers and forwarded

Yes, except this behavior is completely independent from the option "Disable
Standard Mailheaders". It should work in all cases even when that box is not
checked.

Explanation: Once you use the %sig% variable, the standard signature placement
from Thunderbird is not used anymore, and we insert the signature at the place
where you want it. That's why I did not understand the bug under its original
title.

And having it as a global option only might be counter-productive as a lot of
people see signature as something that is account-specific. E.g. they might use
a different signature for newsgroups, for their personal and for their work
email. It will take a while to develop this new option as it will need input
from our translators, once we have the next version released, I will create
some test versions and append them to this bug.

This latest beta works for me - if you want to disable the gray font at the
end, use %sig(2)% or use a html signature and put in some explicit font
coloring.

e.g.

verdana; color: #444; box-shadow: 4px 4px 9px -2px rgba(0,0,0,0.65);
border-radius: 1em; padding: 0.4em 2em; border: 1px dashed #444; background:
rgb(230,240,163);
>
<b class="myName" style="text-shadow: 1px 1px 2px #DDD;cursor:pointer;">John
Doe</b>
<br>Bug Requester
<br>
</div>
