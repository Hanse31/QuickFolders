Description:   Opened: 2010-09-06 09:50 
Created an attachment (id=6587) [details]
Image of TB Window

I use your add-on to give myself keyboard shortcuts to several folders. I have
it setup so that Ctrl+1 brings me to one folder, Ctrl+2 to another, etc. When I
am in the main window and I select Ctrl+1 I get a new tab with a blank label
when the folder is empty. I attached a screenshot showing an example. In this
situation, instead of a blank label, I would expect the label "temp - SRI
Email" (without quotes).

I would also like to request a feature where the Ctrl+# sequence does not open
a new tab. I was used to that under TB2. Such option would make it easier for
me to navigate TB as I like to use shortcuts to move from folder to folder.

I am using TB 3.1.2 under Windows XP.

Thank you,

Philip

------- Comment #2 From Axel Grude 2010-09-06 13:56:11 ------- 
After debugging this a little closer, I found out that I am actually opening
the new Thunderbird Tab through my own code, as I am calling the same function
onButtonClick which has additional functionality to OPEN A NEW TAB if you click
on a QuickFolder with CTRL. 

Fix will follow shortly.
