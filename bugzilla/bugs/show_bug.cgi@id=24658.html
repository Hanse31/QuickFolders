------- Comment #1 From Axel Grude 2012-01-10 11:26:49 ------- 
You're not the first one to ask :) And I am thinking about this one for a
while, so chances are good.

In its first incarnation, I envisage a Filter mode that can be toggled on via
the options menu (and not some hidden key combination), as I would expect this
to be something deliberate but not happening too often, so I would minimize the
possibility of it being triggered accidentally.

While in this mode, any mail move operation would trigger the "create new
filter from message" dialog and it would prepopulate as follows:

Rules:
1. name - set to name of target folder
2. from - insert Message Sender (what if multiple messages are pulled?(*))
3. action - move to target folder

(*) for that case we would need to filter out our own messages so it would have
to omit any messages with one of my own email addresses.

I think that would cover probably 90% of the use cases for a start.

During Filter Learn mode, the standard icon (cogwheel) is replaced with the
filter icon. This is to signal to the user that any mail move / copy operation
will create a new filter.

3.0 Prerelease 2

Here is a first crack at this, took a good few days to write, and lots of
research on IRC. I decided to add a step before the filter dialog, so the user
can tell QF what kind of filter she likes. 

Notes
=====
This isn't fully localized yet, it will only work in English / German versions.

The "Filter by Tag" option isn't implemented yet.

There is a styling problem with the options button (the collapse doesn't work
as expected, invisible buttons take up space)

I am open to more suggestions for filter templates, but I think these 4 are
probably good enough for release.

Not implemented: opening an existing filter (if one with the same target folder
exists). this is a difficult requirement and might not be supported in the
initial version.
------- Comment #6 From Axel Grude 2012-01-18 02:33:11 ------- 
Created an attachment (id=7150) [details]
Filter Template Selection

After doing a drag+drop in filter mode, the user has to select from one of the
templates. On clicking Ok, the Edit Filter dialog is presented prefilled with
values from the dragged email, according to the template.

On clicking Cancel, no Filter will be created - the drag and drop operation
itself will not be rolled back (not tested yet).
