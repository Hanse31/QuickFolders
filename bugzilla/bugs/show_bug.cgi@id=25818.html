Description:   Opened: 2014-07-26 04:04 
I have a login where the name of the password field changes from time to time.
now the field is: "password_522279708", but the next time the name will change,
"password_" will stay the same, but the rest after "_" will change. I'd like to
be able to edit this so that it automatically recognizes the field and I don't
have to manually retrieve the password. A feature.

Thanks.
Sorry, English is not my native language.
------- Comment #1 From Axel Grude 2014-07-26 14:26:19 ------- 
While it would be relatively easy to search the page for another hidden text
(password) field if the original password field can't be filled this also opens
up a major security risk: 

If a malicious script hides the original password field by placing another
textbox above it, it would be dangerous if QP would simply fill that box as it
might pass it on in an unwanted way.

However, I am not completely against such a feature as long as we can make sure
that the user is properly notified - e.g. a red notification box at the top of
the page; and the feature would be disabled by default so as to not open up a
security hole by default.

I thought of that as well, but it would complicate it for me as I would have to
store this as a completely new field with the password information - I would
first have to look into how to do this; so initially it might be easier if it
was a global setting.

> It wouldn't be a completely new field.
> Only the last bit of the name would be different.
> So that I can edit the field name and leave the first part intact, which always
> stays the same while adding an Asterix for example for the part that is

Yeah, obviously you do not want to have to "edit" it every time as this
completely defeats the purpose. You do know however that (in Firefox) you
already get the automatic notification that prompts you to use the context
menu? Once you use the context menu to fill the password field you will see a
notification at the top that will updated the field name for you - it is easy
to miss. Click ok and the new password field name will be stored.

Of course this won't help you if the box is randomized at each visit, as it
will always be wrong :)
