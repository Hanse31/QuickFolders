Description:   Opened: 2012-01-28 19:32 
I just installed this software and I think it's a great idea and should help
overall productivity and ease.   One think I did observe is that there is no
specific support for differentiating folders that are from different email
accounts.  While hovering over the tab does identify the originating account I
think a nice feature would be to have the ability (optional of course) to place
the tabs on separate lines or have some other means of identifying or grouping
tabs by email account.

Case in point, if a user has folders with the exact or similar names (like the
normal Inbox, Draft, etc.) in multiple email accounts one must hover over the
tab to determine from which account the folder is being selected.  Even a
visual indicator like a colored dot or hash mark that is automatically placed
on the tab that is associated to each individual email account would suffice.  

There may be a way of doing this now but it wasn't obvious to me as a new user.

Thank you.
------- Comment #1 From Tony Mechelynck 2012-01-28 21:59:34 ------- 
I think the RFE is valid.

In the meantime you can rename tabs, giving different display names to tabs for
(let's say) the Inbox for different accounts. Or you could give them different
colours. You still see the full tooltip on mouseover, i.e. foldername @ Account
Name [server]

To do that, right-click a tab, then select the submenu "QuickFolder commands
▶" then either "Tab color ▶" or "Rename tab…". It's not automatic of
course, you have to select the new name or the new colour yourself; at least it
does persist from one run of the mailer to the next.
------- Comment #2 From Axel Grude 2012-01-29 04:56:32 ------- 
I don't think I will add a feature that supports automatic visual distinction
between accounts (other than the tooltip text) as it would complicate the tool
too much and also would add too much visual clutter. 

Of course I second Tony's comments on that you could rename the tabs or use
color coding (or icons, via userChrome.css) to distinguish them more easily; 
however this is still a valuable suggestion for the (batch)re-coloring of tabs,
as I could imagine a feature that would recolor a group of tabs based on
account. 

I am already planning such a feature based on category (which would change all
tabs of the same category when selecting a color), so I could imagine adding a
mode "based on account". I will keep the bug open until I get around to
re-looking at the coloring. 

Likely, there will be a global coloring context menu (not one per tab) which
will turn the mouse cursor into Paint-brush for pre-coloring one or many tabs.
2 more modes would be per category / account, but I haven't got a UI concept
for this just yet.

Although not automatic I've decided to use the rename capabilities until some
other way is made available.   A possible way of doing the UI is to create
(automatically) a new tab line for each account with the heading "QuickFolders
for [account name]" and then group all of the tabs on the appropriate line.   

I realize that for a low number of tabs this would be a waste of a line but if
it were somehow controllable then one would not select the option until there
were either a sufficient number of tabs of didn't care about the wasted
line(s). Another visual option would be to change the shape of the corners of
the tabs (square, rounded, angle cut, ...) for each account.  That would be
somewhat limited as you would probably run out of shapes but a way around that
would be to make them re-useable on a rotating basis (square, rounded, angle
cut, square, ...).   This would probably only be useable with accounts grouped
on new lines.   Of course if one has that many email accounts then he/she
probably has other problems.  Just some thoughts.

Regardless, thanks for a great add on and your suggestions.   Works great for
me.
