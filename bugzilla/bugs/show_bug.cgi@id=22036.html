Hi Craig - here is a new test version which adds time stamps to the debug
output in the Error Console. 
Example:

QuickFolders 12:54:27.453  [16 ms]   
qfFolderListener: TotalUnreadMessages


after the QuickFolders label it displays the current time, then the elapsed
milliseconds in brackets - counted from the previous debug message. So if you
find a particular high value around 4000 to 5000 ms, then most likely it is the
operation logged _before_ that takes that long. This will be helpful for me to
determine where all that lag time actually goes :)

thanks for your help - just post the part of the debug output that you suspect
is relevant to the bug...

Install and enable QF debug option.

checked debug output with debug option enabled.
