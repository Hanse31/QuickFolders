Created an attachment (id=7101) [details]
A better patch

Here is a big set of fixes for the user defined colors. This mainly relates to
the "Flat Style" tabs, and will render the specified font / background colors
where the following conditions are met:

1. the tab has no specific color selected
2. the tab has a specific color selected but is in "striped" state (color only
covers the top of the tab)

in these cases, the active, inactive and hover color and backgrounds should now
be displayed correctly. Exception: if tab color [x]transparent is selected, the
background will only show the selected color slightly.

There is also a fix for the transparent background and the selected background
color for the Toolbar itself. Works across all selected QF themes.
