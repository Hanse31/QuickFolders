recent folders options

You can't currently shorten it, but you can hide the label - see the
screenshot. By right-clicking the Recent Folders Option on the "General" Tab of
QuickFolders Options a window will open with the about:config settings for the
recent folders tab. I am going to document the options in the next comment.

The reason I am not exposing all these options as part of the dialog is that I
wanted to streamline it  and not confuse the users with a huge amount of
options because this can often lead to important things to be overseen. Also I
would have to localise all these options, and QuickFolders (with currently 414
strings to translate) is already a heavy-weight for any new translator. I might
still change my mind and a build a proper UI in a separate dialog for these
options. 

In this case I am just lazy and use the built in config editor with a "fixed"
search string. If you get a warning about "voiding your warranty" you can
ignore that as it is just a Joke that comes directly from Thunderbird - I might
yet bypass that one as well, as with the fixed search string, you cannot do
much damage here anyway.

Here is a complete description of all options that can modify Recent Folders
behavior:

extensions.quickfolders.recentfolders.showLabel
  toggle this to "false" to hide the label; (default: true)

extensions.quickfolders.recentfolders.folderPathDetail
  0 - Folder Name
  1 - Folder Name - Mail Account
  2 - Account - Folder Path
  3 - Folder Path   (default)
  4 - Folder URI

extensions.quickfolders.recentfolders.maxPathItems
  the maximum number of folders + parent folders when path is shown (default:
3)

extensions.quickfolders.recentfolders.itemCount
  maximum number of items in the recent folders list (default:12)

extensions.quickfolders.recentfolders.showTimeStamp
  show the time stamp for the folder which this list is normally sorted by 
(default: false)
  I introduced this recently more for testing purposes. The timestamp is also
displayed if the debug setting extensions.quickfolders.recentFolders.detail is
set; I might change this now that I have the option above.

extensions.quickfolders.recentfolders.color
  the palette entry for the background color (default: 10) - normally yellow

extensions.quickfolders.recentfolders.showIcon
  display the "clock" icon (default: true)

extensions.quickfolders.recentfolders.sortAlphabetical
  instead of sorting the recent list by date descending, sort by label
alphabetically

I might yet build a separate UI for the more important ones of these...

ONe more noteworthy fact: you can also display the recent folders button on the
current folder toolbar (center of screen) instead and just hide it on the main
toolbar - use the Advanced Tab / Current Folder Toolbar options list for this.
You can then use the envelope beside it to drag the currently selected email(s)
into a recent folder without much mouse effort. In traditional 3-pane view, the
location of the current folder toolbar ensure minimal mouse mileage for all
your navigation / mail management needs.

I will change the bug status to "LATER" because I believe hiding the label
should probably already fit your needs of saving space if that's okay?
